Títulos

Títulos possuem importância semantica para interpretação exclusivamente dos navegadores e do mecanismos de busca. Principalmente os mecanismos de busca e isso funciona da segunite forma:

Caso um usuário pesquise por um palavra chave x e seu título h1 contenha ou seja exatamente essa palavra chave o mecanismo irá rankear melhor sua página nos resultado que uma página que possua essa palavra chave em um h2 ou outros.

E se essa palavara chave estiver apenas numa tag "p" provavelmente sua página nem aparecerá na pesquisa

Tag semânticas e não semânticas

Existem muitas tag que não possuem efeito semântico algum apenas visual e temo algumas semânticas que possuem efeito visual semelhante. Mas o grande PQ da questão é que qd usamos uma tag semântica o mecanismo de busca entende que aquela palavra tem um significado maior para aquele contexto, logo ele levará em consideração ela na hora de ranker o site nas pesquisas.

Já as tag não semântica não recebem atenção do mecanismos

Quando o navegador inicia ele começa a ler o código html por etapa:

  1. Lê o head, na ordem um linha por vez
  2. Lê o body também na ordem e uma linha por vez

Por isso é importante o link para o arquivo CSS está sempre no head pois assim ele pausa o carregamento da página, carrega todos os estilos e depois volta para o html.

<!DOCTYPE html>

DOCTYPE não é um elemento html, ele apenas informa para o browser que esse html se trata de um html5

lang

Quando criamos um html 5 logo após informar o DOCTYPE ao navegador, nós devemos informar a língua padrão que está sendo usada na página. Por padrão ela vem configurada com "en" de inglês, então sempre devemos muda-la para "pt-br" portugês brasil

Isso serve para que mecanismos de busca indiquem a página para pessoas da linguagem correta. Além disso, informar o navegador o idioma da página, pois caso alguém de outra língua navegue em nossa página o navegador poderá oferecer a opção de tradução correta.

head

head é a parte onde fica toda as configurações iniciais de um html, aqui é onde declaramos configurações que darão infomações importantes para os motores de buscas e entrw outras configurações.

<title>xxx</title>

Title Define o título que será exibido na aba do navegador. Porém muito mais importante que isso é que ele é um dos elementos que possuem maior peso, maior relevância para os mecanismos de busca na hora de renkear uma busca orgânica.

Por isso os title devem sempre ser bem significativo e possuir a palavra chave para rankear a pesquisa no Google. Em outras palavras essa tag faz parte do marketing.

style

O style no head é conhecido como Estilo CSS3 Interno deve ser evitado e usado apenas quando para modificações muito específica daquela página e que nenhuma outra possa ter.

Mais importante ainda essa style no head não pode ser influenciada por estilos externo. Então essa deverá ser um estilo que nao descaracterize a página em relação ao resto do site. Para isso devemos evitar usar algumas propriedade de estilo nesse estilo interno como:

  • Cores
  • Border
  • font-family

metatag

Metatags servem para fazermos as configurações do próprio documento no documento.

charset

Essa configuração meta definhe qual padrão de caracteres usaremos em nossa página, é como definir qual será a biblioteca usada para identificar o caracteres que usaremos na nossa página. No Brasil devemos usar sempre o UTF-8 pois é o que engloba acentuações, til, c cidilha enfim.

Quando usamos o charset errado o navegador não interpreta esses caracteres e a página fica cheia de erros no texto..

O comando:


    <meta charset="UTF-8">

viewport

Essa meta pega a largura do navegador e adapta o conteúdo do site a essa largura tornando o site mais responsivo.

Esse é um dos principais diferenciais do html5

O comando:


    <meta name="viewport" content="width=device-width, initial-scale=1.0">

description

Essa meta é muito importante para rankear bem noss conteúdo nos mecanismos de busca, pois ela é o que aparece na descrição do nosso site no Google logo abaixo do título

Vale apena estudar um pouco de marketing digital para preencher essa descrição

O comando:


    <meta name="description", content="Tudo que você precisa conhecer sobre tags semântica do html5">

Das 3 metas que vimos até agora essa é a primeira que não vem por padrão quando digitamos"!" para iniciar um doc html 5. Porém ela é uma excelente prática, que dever ser adotada em todo site que criarmos

Resumo aula do curso Gustavo Guanabara

Quando cria-se um site em HTML 5(ou HTML Semântico) não se faz modificação de cores formatos de textos ou efeitos visuais utilizando o HTML, esse tipo de efeitos/embelezamento da página é feito usando o CCS que veremos posteriormente.

Mas por que disso?

Isso é feito pra ter funcionalidades e não como apenas leitura, pois se deseja destacar algo no texto, é porque aquilo tem algum significado especial logo ele pode ter uma funcionalidade uma praticidade para o usuário final. Essas mudanças do formato antigo é para que o programa também saiba do que se trata cada item.

Por exemplo: quando digitamos um endereço invés de escrever sublinhado com <u>:

rua fernandes leão, 67, vaz lobo , Rio de janeiro, RJ

Nós escrevemos como <address>:

rua fernandes leão, 67, vaz lobo , Rio de janeiro, RJ

Assim um usuário em um smartphone pode simplesmente clicar no endereço, já abrir no Maps e ir para o local.

Por isso os títulos são escritos dentro de <h1>, invés de apenas mudar para negrito e aumanetar ou diminuir a fonte.

Futuramente ao usar CCS ou Java Script poderemos usar essas funções para gerar ações mais intuitivas para o cliente.

Hierarquia de Títulos

A Hierarquiade títulos é a organização de títulos e subtítulos como já vimos anteriormente <h1> é o comando usado para criar título porém existe também <h2> ,<h3> ,<h4> ,<h5> e <h6> que são usados como subtítulos de subtítulos até no máximo a 6ª Hierarquia
