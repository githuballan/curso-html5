Imagens Flexíveis

No Capítulo 06 nós vimos como adicionar imagens no HTML, porém pudemos observar também que a imagem não se adapta ao tamanho da tela como acontece com os textos.

Para isso podemos usar a tag <picture>, Com essa tag é possível adicionar mais de uma imagem com tamanhos diferentes para que essas imagens se alternem de acordo com o tamanho de tela do usuário:

Tag picture

Note que a imagem acima muda conforme altera a largura da tela isso ocorre porque a tag <picture> testa todas as tag <source> de cima para baixo, se alguma der como verdadeira ela exibe aquela imagem caso nenhuma dê verdadeira ela exibe a última imagem. Por isso é importante sempre deixar a última imagem sem estar na tag source, para evitar que não carregue imagem alguma.

Tag source media type

Essa tag trabalha como o comando "se" do Excel e ela se divide em 3 partes:

  • 1ª parte é onde o teste verifica o tamanho da tela do dispositivo esse teste é feito "media="(max-width: 750px)"" nesse caso ela está testando o se a tela tem até 750 pixels de largura. Caso a tela tenha até 750 pixels, ela exibirá a imagem configurada em srcset="imagens/foto-p.png" caso contrário ela não exibirá imagem e a tag picture passará para o próximo teste.

    • Outra forma seria pelo tamanho mínimo da tela. Nesse caso a imagem só seria exibida em telas apartir de 750px de largura e a tag ficaria: "media="(min-width: 750px)""

  • 2ª parte: Essa é a parte onde indicamos qual imagem deverá ser carregada, Porém aqui não funciona o atalho "CTRL + ESPAÇO" então para evitar erro é aconselhável usar a tag <img> → "CTRL + ESPAÇO" → e copiar esse endereço para dentro da segunda parte da tag <source>
    • Outro ponto importante ainda sobre a segunda parte é que aqui não é aceito nomes de arquivos o pastas no caminho que contenham :
    • Espaço
    • Acentuação ou;
    • Letra Maiúscula
  • 3ª parte dessa tag o type type="image/png"> nessa parte temos que colocar a extensão da imagem caso contrário ela irá acusar ERRO.

Veja abaixo como fica o código completo com as duas Tags:


<picture> 

<source media="(max-width: 750px)"  srcset="imagens/foto-p.png" type="image/png"> 

<source media="(max-width: 1050px)" srcset="imagens/foto-m.png" type="image/png"> 

<img src="imagens/foto-g.png" alt="Foto Teste"> 

</picture>

Tag Figure e figcaption

A tag figure serve para criar uma relação entre a tag figcaption e a imagem. Por enquanto...

figcaption é a tag que define a legenda que ficará embaixo de imagem.

Exemplo:

o comando:

<figure>
    <img src="imagens/foto-p.png" alt="imagem de exemplo">
    <figcaption>Legenda da imagem</figcaption>
</figure>

Imagens responsivas com CSS (Matéria adiantada)

A melhor forma de tornar imagens responsivas é difinindo a style com:
  • max-width: 100%; Pode ficar definido na folha de estilo geral do site;
  • max-height:(altura da imagem) deve ser difinido individualmente na própria imagem
  • strong

Dessa forma a imagem cresce até o height máxima pra ela não destorcer e o max-width faz ela reduzir de acordo com o tamanho de tela.

Se colocar o height fixo quando a tela for menor a imagem vai ficar esticada.

O max-width: 100%; Pode ficar definido na folha de estilo geral do site, mas o max-height deve ser difinido individualmente na própria imagem

Centralização de imagens

Para centralizar imagens em CSS é necessário aplicar display: block; e depois margin: auto;

Adição de Áudios no HTML

No HTML 5 são aceitos 3 formatos de áudios:

  • MP3: Excelente formato, bem conhecido, lido por muitos navegadores e bem compacto.
  • OGG:Pouco conhecido, lido por alguns navegadores e muito compacto
  • WAV: Muito conhecido, lido por muito navegadores, porém muito pesado. Esse formato deve ser usado em último caso.

Tag audio:

Na primeira parte dessa tag nós indicaremos o local e que bom que essa tag aceita o Atalho "ctrl+espaço". Então a primeira parte fica:

"<audio src="ctrl+espaço"...

Atributos

Atributo autoplay: assim o aúdio começa a tocar assim que a página for carregada. PORÉM, usuário não suportam isso pois se o volume estiver alto pode assustar e o usuário sairá da página imediatamente. Outro ponto negativo é que a maioria dos navegadores ignoram esse comando e não reproduzem o audio. E o pior ponto desse Atributo é que a página só termina de carregar quando o áudio estiver concluído o download, o que deixará a página extremamente pesada.
Atributo Control: Esse Atributo exibe os controles básicos do áudio disponbilizado pela página como na imagem abaixo NUNCA ESQUEÇA ESSE DE ADICONAR ESSE ATRIBUTO :

Atributo loop: Faz o áudio ficar reproduzindo em loop, quando acabar o áudio recomeça até o usuário pausar

Atributo Preload: nesse Atributo podemos escolher que dados do áudio serão carregados juntos com a página:
"<audio src="ctrl+espaço" controls preload"auto"..."
Auto: Carrega todo o áudio. Não é indicado pois demora mais a carregar a página e se preencher nada o auto é o padrão, então preencha!
Metadata: Carrega apenas algumas informações sobre o áudio como: duração, nome e formato. esse é INDICADO!
"<audio src="ctrl+espaço" controls preload"metadata"..."
None:Carrega nada.
"<audio src="ctrl+espaço" controls preload"none"..."

Tag áudio como tag picture

A tag áudio também pode funiconar semelhante a tag picture onde ela escolhe qual audio carregar. Esse Atributo serve para evitar incompatibilidade com o formato do áudio. Assim podemos botar na ordem de preferência audios em foramtos mais leves e caso o navegador não suporte ele tentará carregar o próximo áudio e assim por diante. E caso nenhum formato seja carregado, poderemos ainda, expor uma mensagem informando que não foi possível carregar o áudio e disponibilizar um link para download, se for o caso.
Para isso basta digitar a tag áudio normal e apagar a parte marcada "<audio src="ctrl+espaço"....
Após isso digite entre as tags, como na ilustracão abaixo, source e selecione source src.
<audio>"source: src"</audio>
Ficará assim:
<audio><source src="ctrl+espaço" type="tipo do arquivo/formato"></audio>
Em src basta usar o atalho ctrl+espaço e selecionar o audio desejado como já vimos antes
Em Type tem vários formatos que são aceitos, é o caso dar um pesquisada no google caso de dúvidas mas basicamente você deve digitar "audio/a extenção do arquivo de áudio sem o ponto"
Exemplo:
<audio><source src=""modulo-01/capitulo-11/audios/Dead Wrong - Jeremy Blake.mp3"" type="audio/mpeg"></audio>

A tag completa fica seguinte forma: 

            <audio preload="metadata">
            <source src="modulo-01/capitulo-11/audios/Exemplo ogg.ogg" type="audio/ogg">
            <source src="modulo-01/capitulo-11/audios/Exemplo mp3.mp3" type="audio/mpeg">
            <source src="modulo-01/capitulo-11/audios/Exemplo wav.wav" type="audio/wav">
            <p> Infelizmente não foi possível carregar essa áudio
            <a href="modulo-01/capitulo-11/audios/Exemplo mp3.mp3">Clique aqui</a>
            para fazer o download</p>
            </audio>

Vídeos

Direitos autorais

Assim como imagens e áudios, víedeos também possuem direitos autorais e um bom site para conseguir videos livre de direitos autorais é o Pexels, o mesmo que vimos em imagens.

Formatos de Vídeos para HTML

A edição de vídeos é de suma importância assim como para as fotos. Talvez até mais importante pois um video muito pesado estraga completmente a experiência final do usuário.

Durante o curso utilizaremos o editor de videos gratuito HandBreak, mas existe a possibilidade de fazer conversão em site de forma gratuita.Os principais formatos aceitos pelo navegadores são:

  • mp4;
  • m4v;
  • webm e;
  • ogv (menos compatível porém boa compactação e qualidade)

Adição de Vídeos no próprio site

Obs.: Os servidores de hospedagens de site cobram por tráfego de dados logo, se hospedar um vídeo pesado no próprio site, esse vídeos encarecerá absurdamente o site.

Para adicionar o vídeo no próprio é bem similar a adição de áudios.

Tag video

Essa tag também permite usar a tag source dentro dela assim como audio para que teste todos os formatos do mais leve para o mais pesado.

Porém também podemos usar o source media como nas imagens para definirmos também a resolução do video de acordo com o tamanho da tela, e ainda organizar dentro dos formatos. O problema é que não dá para colocar poster para cada resolução. Outro problema é que caso o usuário esteja com smartphone em pé e deite ele durante a navegação, a resolução não aumentará pois ele precisará atualizar a página com o smartphone deitado.

Atributos da tag video

control:

Essa tag é de extrema importância pois ela exibe os controles de vídeos no navegador, assim como no áudio, se não adicionar essa atribuição o cliente não conseguirá dar plays no vídeo.

width

Essa atribuição configura a largura padrão que o vídeo será exibido independente da largura de tela.

poster

Essa permite exibir uma imagem inicial do vídeo invés do primeiro frame do vídeo.

loop

Assim como no áudio o loop faz o vídeo repetir infinitamente até o usuário pausar.
