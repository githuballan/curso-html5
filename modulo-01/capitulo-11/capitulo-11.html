Imagens Flexíveis

No Capítulo 06 nós vimos como adicionar imagens no HTML, porém pudemos observar também que a imagem não se adapta ao tamanho da tela como acontece com os textos.

Para isso podemos usar a tag <picture>. Com essa tag é possível adicionar mais de uma imagem com tamanhos diferentes para que essas imagens se alternem de acordo com o tamanho da tela do usuário:

Tag picture

Note que a imagem acima muda conforme se altera a largura da tela. Isso ocorre porque a tag <picture> testa todas as tags <source> de cima para baixo. Se alguma der como verdadeira, ela exibe aquela imagem. Caso nenhuma dê verdadeira, ela exibe a última imagem. Por isso é importante sempre deixar a última imagem sem estar na tag source, para evitar que não carregue imagem alguma.

Tag source media type

Essa tag funciona como o comando "se" do Excel e ela se divide em 3 partes:

  • 1ª parte é onde o teste verifica o tamanho da tela do dispositivo esse teste é feito "media="(max-width: 750px)"" nesse caso ela está testando o se a tela tem até 750 pixels de largura. Caso a tela tenha até 750 pixels, ela exibirá a imagem configurada em srcset="imagens/foto-p.png" caso contrário ela não exibirá imagem e a tag picture passará para o próximo teste.

    • Outra forma seria pelo tamanho mínimo da tela. Nesse caso a imagem só seria exibida em telas apartir de 750px de largura e a tag ficaria: "media="(min-width: 750px)""

  • 2ª parte: Essa é a parte onde indicamos qual imagem deverá ser carregada, Porém aqui não funciona o atalho "CTRL + ESPAÇO" então para evitar erro é aconselhável usar a tag <img> → "CTRL + ESPAÇO" → e copiar esse endereço para dentro da segunda parte da tag <source>
    • Outro ponto importante ainda sobre a segunda parte é que aqui não é aceito nomes de arquivos o pastas no caminho que contenham :
    • Espaço
    • Acentuação ou;
    • Letra Maiúscula
  • 3ª parte dessa tag o type type="image/png"> nessa parte temos que colocar a extensão da imagem caso contrário ela irá acusar ERRO.

Veja abaixo como fica o código completo com as duas Tags:


<picture> 

<source media="(max-width: 750px)"  srcset="imagens/foto-p.png" type="image/png"> 

<source media="(max-width: 1050px)" srcset="imagens/foto-m.png" type="image/png"> 

<img src="imagens/foto-g.png" alt="Foto Teste"> 

</picture>

Tag Figure e figcaption

A tag <figure> é usada para agrupar elementos de mídia, como imagens, com suas respectivas legendas, utilizando a tag <figcaption>.

Exemplo:

O código HTML correspondente é:

    <figure>
        <img src="imagens/foto-p.png" alt="Imagem de exemplo">
        <figcaption>Legenda da imagem</figcaption>
    </figure>

Imagens responsivas com CSS (Matéria adiantada)

A melhor forma de tornar imagens responsivas é difinindo a style com:
  • max-width: 100%; Pode ficar definido na folha de estilo geral do site;
  • max-height:(altura da imagem) deve ser difinido individualmente na própria imagem
  • strong

Dessa forma a imagem cresce até o height máxima pra ela não destorcer e o max-width faz ela reduzir de acordo com o tamanho de tela.

Se colocar o height fixo quando a tela for menor a imagem vai ficar esticada.

O max-width: 100%; Pode ficar definido na folha de estilo geral do site, mas o max-height deve ser difinido individualmente na própria imagem

Centralização de imagens

Para centralizar imagens em CSS é necessário aplicar display: block; e depois margin: auto;

Áudios

Direitos autorais

Assim como vimos no capítulo 06 sobre direitos autorais de imagens, com áudios também não é diferente. Uma forma de baixar áudios ou músicas livres de direitos autorais é usando uma ferramenta do YouTube. Para acessá-la basta ir no YouTube → Conta → YouTube Studio → Biblioteca de áudios.

Aqui você poderá fazer downloads de todas as músicas de graça. No entanto, nem todas são 100% livres de direitos autorais, pois algumas exigem atribuição, ou seja, é necessário colocar uma legenda atribuindo aquela música ao autor. Para evitar isso basta filtrar os resultados clicando em Atribuições → Atribuição não necessária.

Adição de Áudios no HTML

No HTML 5 são aceitos 3 formatos de áudios:

  • MP3: Excelente formato, bem conhecido, lido por muitos navegadores e bem compacto.
  • OGG: Pouco conhecido, lido por alguns navegadores e muito compacto.
  • WAV: Muito conhecido, lido por muitos navegadores, porém muito pesado. Esse formato deve ser usado em último caso.

Tag audio:

Na primeira parte dessa tag nós indicaremos o local e que bom que essa tag aceita o atalho "ctrl+espaço". Então a primeira parte fica:

"<audio src="ctrl+espaço"...

Atributos

Atributo autoplay: assim o áudio começa a tocar assim que a página for carregada. PORÉM, os usuários não suportam isso pois se o volume estiver alto pode assustar e o usuário sairá da página imediatamente. Outro ponto negativo é que a maioria dos navegadores ignora esse comando e não reproduz o áudio. E o pior ponto desse atributo é que a página só termina de carregar quando o áudio estiver concluído o download, o que deixará a página extremamente pesada.
Atributo controls: Esse atributo exibe os controles básicos do áudio disponibilizado pela página como na imagem abaixo NUNCA ESQUEÇA DE ADICIONAR ESSE ATRIBUTO :

Atributo loop: Faz o áudio ficar reproduzindo em loop, quando acabar o áudio recomeça até o usuário pausar.

Atributo preload: nesse atributo podemos escolher que dados do áudio serão carregados juntos com a página:
"<audio src="ctrl+espaço" controls preload="auto"..."
Auto: Carrega todo o áudio. Não é indicado pois demora mais a carregar a página e se não preencher nada o auto é o padrão, então preencha!
Metadata: Carrega apenas algumas informações sobre o áudio como: duração, nome e formato. Esse é INDICADO!
"<audio src="ctrl+espaço" controls preload="metadata"..."
None: Não carrega nada.
"<audio src="ctrl+espaço" controls preload="none"..."

Tag áudio como tag picture

A tag áudio também pode funcionar semelhante à tag picture onde ela escolhe qual áudio carregar. Esse atributo serve para evitar incompatibilidade com o formato do áudio. Assim podemos colocar na ordem de preferência áudios em formatos mais leves e caso o navegador não suporte ele tentará carregar o próximo áudio e assim por diante. E caso nenhum formato seja carregado, poderemos ainda, expor uma mensagem informando que não foi possível carregar o áudio e disponibilizar um link para download, se for o caso.

Para isso basta digitar a tag áudio normal e apagar a parte marcada "<audio src="ctrl+espaço"....

Após isso digite entre as tags, como na ilustração abaixo, source e selecione source src.

<audio>"source: src"</audio>

Ficará assim:

<audio><source src="ctrl+espaço" type="tipo do arquivo/formato"></audio>

Em src basta usar o atalho ctrl+espaço e selecionar o áudio desejado como já vimos antes.

Em type tem vários formatos que são aceitos, é o caso de dar uma pesquisada no Google em caso de dúvidas mas basicamente você deve digitar "audio/a extensão do arquivo de áudio sem o ponto".

Exemplo:

<audio><source src="modulo-01/capitulo-11/audios/Dead Wrong - Jeremy Blake.mp3" type="audio/mpeg"></audio>

A tag completa fica da seguinte forma:

        <audio preload="metadata">
        <source src="modulo-01/capitulo-11/audios/Exemplo ogg.ogg" type="audio/ogg">
        <source src="modulo-01/capitulo-11/audios/Exemplo mp3.mp3" type="audio/mpeg">
        <source src="modulo-01/capitulo-11/audios/Exemplo wav.wav" type="audio/wav">
        <p> Infelizmente não foi possível carregar esse áudio
        <a href="modulo-01/capitulo-11/audios/Exemplo mp3.mp3">Clique aqui</a>
        para fazer o download</p>
        </audio>

Vídeos

Direitos autorais

Assim como imagens e áudios, vídeos também possuem direitos autorais e um bom site para conseguir vídeos livres de direitos autorais é o Pexels, o mesmo que vimos em imagens.

Formatos de Vídeos para HTML

Os principais formatos aceitos pelos navegadores são:

  • MP4: Formato mais popular, compatível com a maioria dos navegadores e dispositivos.
  • M4V: Similar ao MP4, usado principalmente pela Apple.
  • WebM: Formato aberto e eficiente, suportado por navegadores modernos.
  • OGV: Menos compatível, mas oferece boa compactação e qualidade.

Durante o curso utilizaremos o editor de vídeos gratuito HandBrake, mas existe a possibilidade de fazer conversão em sites de forma gratuita.

Adição de Vídeos no próprio site

Obs.: Os servidores de hospedagem de sites cobram por tráfego de dados, logo, se hospedar um vídeo pesado no próprio site, esse vídeo encarecerá absurdamente o site.

Para adicionar o vídeo no próprio site é bem similar à adição de áudios.

Tag video

Essa tag também permite usar a tag source dentro dela assim como audio para que teste todos os formatos do mais leve para o mais pesado.

Porém também podemos usar o source media como nas imagens para definirmos também a resolução do vídeo de acordo com o tamanho da tela, e ainda organizar dentro dos formatos. O problema é que não dá para colocar poster para cada resolução. Outro problema é que caso o usuário esteja com smartphone em pé e deite ele durante a navegação, a resolução não aumentará pois ele precisará atualizar a página com o smartphone deitado.

Principais atributos da tag video:

  • controls: Exibe os controles de reprodução do vídeo (play, pause, volume, etc.).
  • autoplay: Faz o vídeo começar a reproduzir automaticamente assim que a página é carregada.
  • loop: Faz o vídeo reiniciar automaticamente quando termina.
  • muted: Inicia o vídeo sem som.
  • poster: Define uma imagem a ser exibida antes do vídeo começar a ser reproduzido.
  • preload: Indica ao navegador como deve tratar o carregamento do vídeo. Pode ser:
    • auto: Carrega todo o vídeo quando a página é carregada.
    • metadata: Carrega apenas os metadados do vídeo (como duração, dimensões, etc.).
    • none: Não carrega nada do vídeo até que o usuário inicie a reprodução.
  • width: Define a largura do vídeo.
  • height: Define a altura do vídeo.

Sintaxe:

            <video controls autoplay loop muted poster="caminho/para/imagem.jpg" preload="auto" width="640" height="360">
                <source src="caminho/para/video.mp4" type="video/mp4">
                <source src="caminho/para/video.webm" type="video/webm">
                Seu navegador não suporta a tag de vídeo.
            </video>

Adição de vídeos de sites Externos

YouTube

MELHOR OPÇÃO basicamente carregue o vídeo no YouTube, depois abra o vídeo no YouTube, clique em compartilhar → incorporar → copiar → cole no local desejado no VS Code.

Essa é a melhor opção porque não consome tráfego nem dados da sua hospedagem. Em resumo, vídeos gratuitos sem custos adicionais.

Ponto negativo, ao final do vídeo o youtube oferece outra opções de vídeo e nessas opções pode acabar aparecendo uma opção de um concorrente.

Vimeo

Outra opção de hospedagem de vídeos é o Vimeo. Nesse site, você pode hospedar seus vídeos com mais qualidade, sem exibir vídeos ao final e sem limites de transferência de dados.

Ponto negativo: é um serviço pago e possui limite de vídeos.
