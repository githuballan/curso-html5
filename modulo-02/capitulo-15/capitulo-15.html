Trabalhando com id e class

id são identificadores e class são classificadores. Eles servem para darmos estilos prórpio para cada item de acordo com suas característica exclusiva.

Algumas OBS.:

  • id em HTML é: id="nome-do-identificador".
  • id em CSS é: #nome-do-identificador
  • class em HTML é: class="nome-da-classe"
  • class em CSS é: .nome-da-classe
  • Comentar em HTML: <!-- Comentário-->
  • Comentar em CSS: /*Comentário*/

id→#

O "id" nada mais é que um identificador. Essa função é utilizada para identificarmos algo específico na página. Por exemplo o Título dessa página atual é Capítulo 15 e está em h1, porém só existe um título assim que nomeia a página inteira, mas existem muitos outros títulos h1 na página logo podemos identificar esse h1 do Capítulo 15 com id. Assim poderemos dar uma personalização a mais.

Exemplo no HTML:

<h1 id="capitulo">Capítulo 15</h1>

Exemplo no CSS3:


#capitulo
{
font-family: Georgia, 'Times New Roman', Times, serif;
font-size: 2.5em;
background-image: radial-gradient(circle, #567eb942, #bad5dd);
}

OBS.: Se houver personalização conflitante entre o h1 e o id o CSS vai sempre priorizar a personalização de id.

OBS.: Segundo o W3C Não podemos usar mais de um id no mesmo HTML por mais que funcione, isso não está correto.

Atalho:
Para selecionar mais de um parte do texto ao mesmo tempo. Basta seguras a tecla "ALT" e clicar ou selecionar com o mouse os locais que deseja editar simultaneamente.

class→.

Quando necessário identificar mais de um elemento com a mesma personalização em um mesmo doc HTML ele se torna uma "class".
Exemplo no HTML: 

<p class="mark"></p>

Exemplo no CSS3: 
            .mark
                {
                    background-color: yellow;
                }

Mais de uma class no mesmo item

É possível adicionar mais de uma class no mesmo item. Ficaria da seguinte forma:
Exemplo no HTML: 

<p class="mark mark2"></p>

Exemplo no CSS3: 
            .mark
                {
                    background-color: yellow;
                }
                .mark2
                {
                    font-size: 2.5em;
                }

Também é possivel adicionar id e class no mesmo item. Mas nesse caso as configurações do id se sobrepõe caso haja conflito com a class.

O que NUNCA fazer!

Nunca devemos nomear a class com a caracterísca que ela gera para o elemento. Pois quando se faz isso você perde toda a Semântica do seu HTML.
Outro ponto é que, se por acaso quiser mudar a personalização da class um dia. Provavelmente irá acontecer algo do tipo: Exemplo no CSS3: 

            .bgvermelho
                {
                    background-color: yellow;
                }
                .fontgrande
                {
                    font-color: #7bd768;
                }
Note que o exemplo acima ficou uma zona!

Atualização da Hierarquia de Estilos

Se houver personalizações conflitantes, a hierarquia de prioridades entre elas fica da seguinte forma:

  1. Estilo InLine
  2. id
  3. class
  4. Estilo CSS3 Interno
  5. Estilo CSS3 Externo

Pseudo-classes→:

Uma pseudo-classe CSS é uma palavra-chave adicionada às declarações de um seletor após um sinal de dois pontos (:). Basicamente geram uma dinâmica no site, deixa ele mais intuitivo e mais interativo.
Alguns exemplos conhecidos são passar o mouse e abrir uma aba (:hover). Existem as pseudo classes nativas como as de link onde ela ficam roxas depois de visitado (:visited).
Veja abaxo alguns exemplos de pseudo classes:

  • :hover: Gera uma reação quando o mouse passar em cima;

    Cursor: Essa pseudo classe costumar ser muito usada com o estilo de cursor. Para quando passar o mouse em cima de algum elemento, mudar o cursor dependendo da funcionalidade que estiver disponível. Para saber mais sobre essa propriedade acesse o site da MDN

  • :visited: Modifica a forma de um link que já foi visitado;
  • :link: Adiciona um estilo especial a um vínculo não visitado ;
  • :active: Adiciona um estilo especial a um vínculo selecionado(clicado);
  • :focus: deixa marcado o último item clicado;
  • :lang:Permite que o autor especifique uma linguagem a ser usada num elemento específico;

Pseudo-elementos →::

Pseudo-elementos são acessório que ficam envolta do elemento principal. É como um personalização, porém invés de modificar cores ou fontes. Essa personalização insere novos elementos como uma seta ou textos para da destaque, ou alertas, vai depender da sua intensão.

Agora veja exemplos de funcionamento e códigos:

Exemplo de código no head: 
        <style>
                .exafter::after{
                    content: '<<<<';
                    }
            <style>
Exemplo de código no body: 
                <p class="exafter">
                        agora
                    </p>
Exemplo de funcionamento:

agora

Note que não dá para selecionar o after.

Div

Div é uma box não semântica que permite adicionar estilos a itens contidos nela. Quase sempre usamos div com alguma classe ou id, pois ela só serve para darmos estilos e se usar apenas div sem class, ela servirá apenas como um conteiner.

Exemplo de código no head:

            
    <style>
        div{
        color: blue;
        }
    <style>
Exemplo de código no body: 

    <div>
        TESTANDO A TAG DIV
    </div>
Exemplo de funcionamento:
TESTANDO A TAG DIV

Children → >

Children serve para se referir uma tag que está dentro da tag div. Para isso usamos o símbolo de maior que (>).

Exemplo de código no head:

                
                <style>
                    div>p{
                        background-color: rgb(255, 255, 000);
                    }
                <style>
Exemplo de código no body: 
                
                    <div>
                        <p> TESTANDO A TAG DIV</p>
                    </div>
Exemplo de funcionamento:

TESTANDO A TAG DIV

Hierarquia CSS

Como já vimos quanto mais específico maior o grau hierarquico do estilo, logo se usar uma children para definir um item essa será a prioridade da CSS.

  1. Inline
  2. Interno
    1. id
    2. class
    3. children
    4. comum
  3. Externo
    1. id
    2. class
    3. children
    4. comum

Mistura de id class e children

Quando mistura children com class e id o conteiner com id é prioridade acima de qualquer conteiner sem id e acima de id ou class da box interna

  1. Conteiner com id
  2. Box interna com id
  3. Conteiner com class
  4. Box interna com class
  5. Conteiner puro
  6. Box interna pura
