Grid-Layout

Propriedades do Container

O Grid Layout é uma técnica relativamente simples que tem por finalidade definir o layout principal da página através de grades que se ajustam de forma responsiva. Diferentemente das flex-boxes, que se comportam como um fluido, a grid trabalha de forma mais rígida, semelhante a uma tabela.

Propriedades básicas

    As propriedades básicas do Grid Layout são fundamentais para criar layouts eficientes e organizados. Elas permitem definir a estrutura da grade, controlando a quantidade de linhas e colunas, bem como o espaçamento entre os elementos. A seguir, exploraremos algumas dessas propriedades essenciais.

  • display: grid; Este comando define que esta caixa é um container que se comportará como grid.
  • grid-template-columns: auto auto auto; Este comando define a quantidade de colunas que este container terá. Neste caso, foram definidas 3 colunas de tamanho adaptável de acordo com a tela. Entretanto, se limita ao conteúdo da caixa, ou seja, quando não for possível reduzir mais seu conteúdo, ela irá gerar um overflow. Por padrão, o container vem definido para uma coluna em "auto".
  • grid-template-rows: 150px; Semelhante ao comando acima, mas este funciona com linhas. Neste caso, como foi colocado apenas um "150px", apenas a primeira linha terá 150px, as demais ficarão como "auto".

    • A diferença é que este, por padrão, assume quantas linhas forem necessárias para a quantidade de caixas que existirem no container.

    Caso defina quatro linhas de 150px e duas colunas "auto", uma quinta linha seria criada abrigando duas caixas com tamanho "auto".

  • grid-template:auto auto / auto auto auto; Essa propriedade finciona como uma abreviação de grid-template-columns: e grid-template-rows: onde o primeiro valor refere-se às colunas e o segundo às linhas. Neste caso, foram definidas 3 colunas e 2 linhas, todas com tamanho "auto".
  • column-gap: Esta propriedade define um espaço padrão entre as colunas de caixas. Ela funciona como a "justify-content: space-between" das flex-boxes.
  • row-gap: Esta define um espaçamento padrão entre as linhas da grid. Ela funciona como a "align-content: space-between" das flex-boxes.
  • gap: Esta é como um shorthand onde podemos definir os dois espaçamentos acima em uma só linha, sendo o primeiro referente à linha e o segundo referente à coluna.

    • Vale ressaltar que, caso defina apenas um valor, este valor será aplicado aos dois espaçamentos.

Grid Template Area

Esta propriedade é uma forma de definir o layout da página de forma mais visual, sem a necessidade de definir quantas colunas e linhas o container terá. Vejamos o exemplo abaixo:

grid-template-areas:
"header header header header"
"nav main main main"
"nav main main main"
"footer footer footer footer";

Este comando define que o container terá 4 colunas e 4 linhas, onde:

  • o header ocupará as 4 colunas da primeira linha,
  • a nav ocupará a primeira coluna das linhas 2 e 3,
  • o main ocupará as 3 colunas restantes das linhas 2 e 3 e
  • o footer ocupará as 4 colunas da última linha.

Mesclar linhas e colunas

Vamos agora falar sobre as propriedades dos itens da grade. Itens são as caixas dentro do contêiner.

  • grid-column-start: 1; Define em qual coluna o item irá começar.
  • grid-column-end: 2; Define até qual coluna o item irá se esticar. Ele não inclui a coluna final, apenas vai até ela. Neste caso, como está até a coluna 2, não mudaria nada. Mas se fosse até a 3, o item se esticaria absorvendo o espaço da coluna 2.
  • grid-column: 1 / 3; Este comando é uma abreviação dos dois anteriores, onde o primeiro número refere-se à coluna onde o item começa e o segundo onde ele termina.
  • grid-row-start: 1; Idêntico ao "grid-column-start", mas neste caso trata-se de linhas.
  • grid-row-end: 1; Idêntico ao "grid-column-end", mas neste caso trata-se de linhas.
  • grid-row: 1 / 3; Idêntico ao "grid-column", mas neste caso trata-se de linhas e não aceita o "span".
  • grid-area: 1 / 1 / 2 / 2; Este comando é uma abreviação que define a área ocupada pelo item. A ordem dos valores é:
    1. grid-row-start
    2. grid-column-start
    3. grid-row-end
    4. grid-column-end
  • span: Esta propriedade permite que o item se estenda por várias colunas ou linhas. Por exemplo, se você quiser que um item se estenda por 3 colunas, basta definir "grid-column-end: span 3;".

    Nesse caso, não importa onde o item inicia, ele sempre se estenderá por mais 3 colunas à frente.

    OBS.: Preste atenção à sintaxe, pois há um espaço entre "span" e o número. Caso não coloque esse espaço, o item se estenderá por todas as linhas ou colunas disponíveis à frente.

Importante: As propriedades que estendem os itens de acordo com as colunas não funcionarão caso a largura (width) da caixa esteja definida. Da mesma forma, as propriedades que estendem os itens de acordo com as linhas não funcionarão se a altura (height) da caixa estiver definida.

Posicionamento da caixa dentro da célula

Os itens dentro de um contêiner de grid podem ser posicionados de várias maneiras usando propriedades específicas. Aqui veremos como posicionar os itens dentro de suas células:

  • justify-items: Alinha o item dentro de sua célula no eixo horizontal. Os valores possíveis são start, end, center e stretch.
  • align-items: Alinha o item dentro de sua célula no eixo vertical. Os valores possíveis são start, end, center e stretch.
  • place-items: É uma abreviação para align-items e justify-items. O primeiro valor é para align-items e o segundo é para justify-items. Se apenas um valor for fornecido, ele será aplicado a ambos.

Posicionamento da tabela dentro do contêiner

Além de posicionar os itens dentro de suas células, também podemos controlar o posicionamento da tabela dentro do contêiner usando as propriedades justify-content e align-content.

  • justify-content: Alinha as células dentro do contêiner no eixo horizontal. Os valores possíveis são start, end, center, space-between, space-around, e space-evenly.
  • align-content: Alinha as células dentro do contêiner no eixo vertical. Os valores possíveis são start, end, center, space-between, space-around, e space-evenly.
  • place-content: É uma abreviação para align-content e justify-content. O primeiro valor é para align-content e o segundo é para justify-content. Se apenas um valor for fornecido, ele será aplicado a ambos.

Autonomia dos itens

Além de alinhar todos os itens dentro de suas células e as células dentro do contêiner, você pode alinhar individualmente cada item em relação à sua célula usando as propriedades justify-self e align-self.

  • justify-self: Alinha o item no eixo horizontal. Os valores possíveis são start, end, center, stretch.
  • align-self: Alinha o item no eixo vertical. Os valores possíveis são start, end, center, stretch.
  • place-self: É uma abreviação para align-self e justify-self. O primeiro valor é para align-self e o segundo é para justify-self. Se apenas um valor for fornecido, ele será aplicado a ambos.
