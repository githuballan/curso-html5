Grid-Layout

O Grid Layout é uma técnica relativamente simples que tem por finalidade definir o layout principal da página através de grades que se ajustam de forma responsiva. Diferentemente das flex-boxes, que se comportam como um fluido, a grid trabalha de forma mais rígida, semelhante a uma tabela.

Container

Assim como em flex-box, quando trabalhamos com grid também temos um container que abriga todas as demais caixas.

Para isso, usamos os seguintes comandos no container:

  • display: grid; Este comando define que esta caixa é um container que se comportará como grid.
  • grid-template-columns: auto auto auto; Este comando define a quantidade de colunas que este container terá. Neste caso, foram definidas 3 colunas de tamanho adaptável de acordo com a tela. Entretanto, se limita ao conteúdo da caixa, ou seja, quando não for possível reduzir mais seu conteúdo, ela irá gerar um overflow. Por padrão, o container vem definido para uma coluna em "auto".
  • grid-template-rows: 150px; Semelhante ao comando acima, mas este funciona com linhas. Neste caso, como foi colocado apenas um "150px", apenas a primeira linha terá 150px, as demais ficarão como "auto".

    • A diferença é que este, por padrão, assume quantas linhas forem necessárias para a quantidade de caixas que existirem no container.

    Caso defina quatro linhas de 150px e duas colunas "auto", uma quinta linha seria criada abrigando duas caixas com tamanho "auto".

  • column-gap: Esta propriedade define um espaço padrão entre as colunas de caixas. Ela funciona como a "justify-content: space-between" das flex-boxes.
  • row-gap: Esta define um espaçamento padrão entre as linhas da grid. Ela funciona como a "align-content: space-between" das flex-boxes.
  • gap: Esta é como um shorthand onde podemos definir os dois espaçamentos acima em uma só linha, sendo o primeiro referente à linha e o segundo referente à coluna.

    • Vale ressaltar que, caso defina apenas um valor, este valor será aplicado aos dois espaçamentos.

Grid-template-areas

Esta propriedade é uma forma de definir o layout da página de forma mais visual, sem a necessidade de definir quantas colunas e linhas o container terá. Vejamos o exemplo abaixo:

grid-template-areas:
"header header header header"
"nav main main main"
"nav main main main"
"footer footer footer footer";

Este comando define que o container terá 4 colunas e 4 linhas, onde:

  • o header ocupará as 4 colunas da primeira linha,
  • a nav ocupará a primeira coluna das linhas 2 e 3,
  • o main ocupará as 3 colunas restantes das linhas 2 e 3 e
  • o footer ocupará as 4 colunas da última linha.

Html: 

        
        <div class="contex" id="conteiner">
            <div class="intex" id="cabeçalho">Cabeçalho</div>
            <div class="intex" id="principal">Principal</div>
            <div class="intex" id="rodape">Rodapé</div>
            <div class="intex" id="lateral">Lateral</div>
        </div>

CSS:

    
.contex{/* Tabela article Alinhamentos nos eixos*/
    display: grid;        
    background-color: coral; 
    padding: 5px;  
    }

.intex{/*exemplos alinhamento dos eixos*/
    background-color: aquamarine;       
    padding: 40px 0px 40px 0px;      
    border-radius: 20px;
    text-align: center;
    border:1px solid #000000;
    min-width: 100px;
    }

#conteiner{          
    display: grid;
    gap: 5px;
    grid-template-columns: auto auto auto auto;
    grid-template-rows: 150px 150px 150px 150px;
    grid-template-areas: 
    "header header header header"
    "nav main main main"
    "nav main main main"
    "footer footer footer footer";
    }

#cabeçalho{grid-area: header;}
div#principal{grid-area: main;}
div#rodape{grid-area: footer;}
div#lateral{grid-area: nav;}

itens

Vamos começar agora a falar sobre propriedade de itens da grade. Itens entende-se as caixas.

  • grid-column-start: 1De forma define em qual coluna esse item irá começar. Use o html testes para melhor compreensão.
  • grid-column-end: 2; Define até qual coluna esse item irá se esticar.Porém ele não inclui a coluna ele vai apenas até. Nessa caso como está até a coluna dois, mudaria absolutamente nada. Mas se fosse até a 3 o item esticaria absorvendo o espaço da coluna 2
  • grid-column: 1 / span 3; Esse comando é como uma shothand dos 2 citados anteriormente onde o primeiro número refere-se a coluna onde o item começa e o sedundo onde ele acaba.

    • Note de nesse caso foi adicionado um span antes do segundo número. Esse span serve apenas para incluir a coluna então nesse caso ele absorve a coluna 3.

    Importante ainda ressaltar que esse span só pode ser usados em coluna, não funciona em rows como veremos mais a frente.

    Ressalto ainda que esse span pode ser aplicado em grid-column-end também e que não é brigatório em grid-column, foi só um exemplo.
  • grid-row-start: 1 Idêntico ao grid-column-start: só que nesse caso trata-se de linha
  • grid-row-end: 1 Idêntico ao grid-column-end: só que nesse caso trata-se de linha
  • grid-row: 1/3; Idêntico ao grid-column: só que nesse caso trata-se de linha e não aceita o span
  • grid-area: 1/1/2/2;Difícil pra caralho PQP. Nesse caso o 1º item permaneceria no lugar inalterado, mas se aplicar essa configuração em qualquer outro item ele tomará o lugar do primeiro item. Mas isso é uma espécie de shothand. A ordem dos itens são:
    1. grid-row-start
    2. grid-column-start
    3. grid-row-end
    4. grid-column-end

Posicionamento

Todas as propriedades de alinhamentos de flex-box funcionam em grid
