Formulários

Formulários é tudo que o usuário preencher no site, como login, senha, pesquisas, comentários entre outros. A tag usada em formulário é a tag form <form></form>. E dentro dessa tag usamos a tag input. que é a tag que faz abre o campo para o usuário digitar.
Veja o exemplo abaixo:

Nome:

Sobrenome:

O Comando: 


<form >

    <p>Nome:<input type="text"></p>
    <p>Sobrenome: <input type="text"></p>
    <p><input type="submit" value="Enviar"></p>

</form>

A primeira e a última linhas vão delimitar o formulário. Tudo aquilo que estiver dentro desse par de tags será considerado como conteúdo dos campos. No exemplo acima, temos apenas 2 campos que vão aceitar um texto relativo ao nome e ao sobrenome da pessoa e um botão de submissão, que vai permitir enviar o formulário.

Porém precisamos fazer algumas configurações adicionais para que esse formulário funcione de maneira mais semântica e correta, para isso adicionaremos alguns atributos:

Atributos para tag form

action

Esse atributo dirá para onde os dados serão enviados e será ativado quando o atributu submit for acionado em <input type="submit" value="Enviar">

Exemplo:

Nome:

Comando: 


<form method="get" action="cadastro.php">
    Nome: <input type="text">        
    <input type="submit" value="Enviar">
</form>

Php é uma das linguagens de programação mais usadas na internet ela é que trata os dados digitados pelo usuário. Temos outras, mas as mais utilizadas são PhP e Java Script.

method

Basicamente, existem dois métodos para enviar os dados de um formulário: GET e POST.

GET

Esse método de envio de dados é o padrão, então se preenche nada esse será o método usado. Para exemplificar o método GET, olhe abaixo o site do Google fazendo uma busca por Curso HTML5.

Depois de enviar o formulário preste bastante atenção à barra de endereços do navegador e veja a URL que você está acessando:

Note que aquilo que você digitou no formulário apareceu na URL que está sendo apresentada no seu navegador. Isso é uma característica presente nos formulários com método GET, pois ele condensa todos os dados digitados pelo visitante e envia como parte do endereço acessado.

Principais características do método GET:

  • Não pode conter dados sensíveis: dados como: endereços, números de documentos, senhas e outros devem ser enviados via método post
  • Simplicidade: Tivermos formulários simples e que possam ser compartilhados facilmente através de uma URL. Por exemplo, o Google usa GET no seu formulário, pois você pode enviar um link para qualquer pessoa, já com a ação da busca embutida, como em http://google.com.br/search?q=Gustavo+Guanabara
  • Máximo 3kb Os dados a serem enviados nunca poderão ultrapassar 3000 bytes.
  • Somente texto: Não contenha envio de arquivos como imagens (i.e. fotos de perfil), arquivos PDF, etc.

POST

Muita gente diz que o método post é o mais seguro. Porém isso não é exatamente verdade, Pois apesar dele não enviar os dados na URL esses dados podem ser facilmente acessados usando a função inspecionar do navegador

Principais características do método POST:

  • Dados não aparecem na URL: Devemos usar esse método quando precisamos enviar dados sensíveis como os citados nas características do get.
  • Aceita envios acima de 3kb: Esse método não possui limites de dados para envio
  • Envia diversos tipos de arquivos

Outra discussão que é bastante levantada nas comunidades é em relação à velocidade ou performance de cada método. De forma resumida, um formulário que usa método GET é ligeiramente mais rápido que um formulário que usa o POST. Porém, não use isso como critério para decidir qual método aplicar ao seu formulário. Foque-se primordialmente nos pontos levantados logo acima e sua escolha será muito mais sábia.

Tag label

A tag label serve para criar uma ligação semântica entre o identificador (nome do caixa de texto) e o caixa de texto(input). Essa tag não possui efeito visual apenas semântico e isso auxilia o macanismo de busca. Em outras palavras é importante para ficar bem poscionado em em buscas do Google.

Essa tag funiciona da seguinte forma:

Comando: 


        <form method="get" action="cadastro.php">
            <label for="lnome">Nome:</label>
            <input type="text" id="lnome">
            <input type="submit" value="Enviar">
        </form>

Note que a palavra "lnome" é o que faz a ligação entre os dois elementos então ela devem ser iguaias e devem ser usada somente uma para cada label.

Resultado:

Note que agora ao clicar em nome automaticamente o cursor vai para a caixa de texto correspondente.

Atributo name

Esse atributo name , também é usado dentro da tag input. Esse atributo serve para identificar a tag input, porém esse name será o título da coluna do formulário que o usuário preencher. Então esse name pode ser exatamente igual ao identificador da caixa de texto. 


        <form method="get" action="cadastro.php">
            <label for="lnome">Nome:</label>
            <input type="text" name="Nome" id="lnome" >
            <input type="submit" value="Enviar">
        </form>

Informção EXTRA

Para desativar o aquela aba de autopreenchimento dos formulários basta adicionar o atributo autocomplete e botar off:

OBS.: o ON vem ativado por padrão.

O comando é simples: 


        <form method="get" action="cadastro.php" autocomplete="off" >
            <label for="lnome">Nome:</label>
            <input type="text" name="Nome" id="lnome"
            <input type="submit" value="Enviar">
        </form>
Resumo de atributo
Tag Atributo Funcionalidade
form action Esse atributo dirá para onde os dados serão enviados e será ativado quando o atributo submit for acionado
method="get" Aparece no URL, não pode conter dados sensíveis, máximo 3kb e só envia texto
method="post" Dados não aparecem na URL, Aceita envios acima de 3kb e envia diversos tipos de arquivos.
autocomplete="off" Desliga o texto que auxilia o usuário (por padrão ele fica sempre ativado)
input type="input" Cria um botão para enviar o formulário
type="reset" Cria um botão para apagar tudo que foi escrito no formulário inteiro
Value="Enviar" Nome que aparecerá no botão. Mas esse input também pode ser usado em caixa de texto, nesse caso ele já aparecerá um texto digitado, é muito usado em data já deixando um data atual pro usuário não ter que correr o calendário inteiro.
type="number" Caixa de texto só aceita número. Note que cria um seta para cima e para baixo dentro da caixa de texto para rolar os números
min="0" O VALOR MÍNIMO para enviar o formulário.
max="10" O VALOR MÁXIMO para enviar o formulário.
step="0.5" Só rola os números de 0,5 em 0,5
type="month" Abre um calendário no navegador para o usuário escolher mês e ano.
type="date" Igual month mas nesse inclui os dias(data completa) É importante sempre usar esse type para validar Informção antes de enviar. Isso evita erros de digitação do usuário.
type="time" Aceita somente horas e minutos
type="password" aparece "*" no lugar do que foi digitado
name="Nome" Esse name será o título da coluna do formulário que o usuário preencher.
required Exige que o formulário esteja preenchido para ser enviado
minlength="5 Exige que o formulário esteja preenchido com no mínimo 5 caracteres para ser enviado
maxlength="15 Exige que o formulário esteja preenchido com no máximo 15 caracteres para ser enviado
size="10" Define o tamanho da caixa para caber até 10 caracteres(a medida padrão da letra X)
placeholder="Digite seu nome aqui" Mostra uma dica dentro da caixa de texto(ela some assim que o usuário começar a digitar na caixa)
autocomplete="username" Guia o navegador para auxiliar o usuário com a sugestão correta, nesse caso seria o nome de Login. Para ver todas as opções basta pressionar"ctrl+espaço" entre as aspas. (Para usar esse autocomplete a tag form deverá estar com o autocomplete="ON")
autocomplete="current-password" O navegador irá sugerir a senha já cadastrada a senha recorrente
autocomplete="new-password" O navegador irá sugerir nada, ou o gerenciador de senha poderá sugerir uma senha forte e salvar nova senha.
