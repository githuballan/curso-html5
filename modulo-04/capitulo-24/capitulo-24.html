Formulários

Formulários é tudo que o usuário preencher no site, como login, senha, pesquisas, comentários entre outros. A tag usada em formulário é a tag form <form></form>. E dentro dessa tag usamos a tag input. que é a tag que faz abre o campo para o usuário digitar.
Veja o exemplo abaixo:

Nome:

Sobrenome:

O Comando: 


<form >

    <p>Nome:<input type="text"></p>
    <p>Sobrenome: <input type="text"></p>
    <p><input type="submit" value="Enviar"></p>

</form>

A primeira e a última linhas vão delimitar o formulário. Tudo aquilo que estiver dentro desse par de tags será considerado como conteúdo dos campos. No exemplo acima, temos apenas 2 campos que vão aceitar um texto relativo ao nome e ao sobrenome da pessoa e um botão de submissão, que vai permitir enviar o formulário.

Porém precisamos fazer algumas configurações adicionais para que esse formulário funcione de maneira mais semântica e correta, para isso adicionaremos alguns atributos:

Tag form

action

Esse atributo dirá para onde os dados serão enviados e será ativado quando o atributu submit for acionado em <input type="submit" value="Enviar">

Exemplo:

Nome:

Comando: 


<form method="get" action="cadastro.php">
    Nome: <input type="text">        
    <input type="submit" value="Enviar">
</form>

Php é uma das linguagens de programação mais usadas na internet ela é que trata os dados digitados pelo usuário. Temos outras, mas as mais utilizadas são PhP e Java Script.

method

Basicamente, existem dois métodos para enviar os dados de um formulário: GET e POST.

GET

Esse método de envio de dados é o padrão, então se preenche nada esse será o método usado. Para exemplificar o método GET, olhe abaixo o site do Google fazendo uma busca por Curso HTML5.

Depois de enviar o formulário preste bastante atenção à barra de endereços do navegador e veja a URL que você está acessando:

Note que aquilo que você digitou no formulário apareceu na URL que está sendo apresentada no seu navegador. Isso é uma característica presente nos formulários com método GET, pois ele condensa todos os dados digitados pelo visitante e envia como parte do endereço acessado.

Principais características do método GET:

  • Não pode conter dados sensíveis: dados como: endereços, números de documentos, senhas e outros devem ser enviados via método post
  • Simplicidade: Tivermos formulários simples e que possam ser compartilhados facilmente através de uma URL. Por exemplo, o Google usa GET no seu formulário, pois você pode enviar um link para qualquer pessoa, já com a ação da busca embutida, como em http://google.com.br/search?q=Gustavo+Guanabara
  • Máximo 3kb Os dados a serem enviados nunca poderão ultrapassar 3000 bytes.
  • Somente texto: Não contenha envio de arquivos como imagens (i.e. fotos de perfil), arquivos PDF, etc.

POST

Muita gente diz que o método post é o mais seguro. Porém isso não é exatamente verdade, Pois apesar dele não enviar os dados na URL esses dados podem ser facilmente acessados usando a função inspecionar do navegador

Principais características do método POST:

  • Dados não aparecem na URL: Devemos usar esse método quando precisamos enviar dados sensíveis como os citados nas características do get.
  • Aceita envios acima de 3kb: Esse método não possui limites de dados para envio
  • Envia diversos tipos de arquivos

Outra discussão que é bastante levantada nas comunidades é em relação à velocidade ou performance de cada método. De forma resumida, um formulário que usa método GET é ligeiramente mais rápido que um formulário que usa o POST. Porém, não use isso como critério para decidir qual método aplicar ao seu formulário. Foque-se primordialmente nos pontos levantados logo acima e sua escolha será muito mais sábia.

Tag label

A tag label serve para criar uma ligação semântica entre o identificador (nome do caixa de texto) e o caixa de texto(input). Essa tag não possui efeito visual apenas semântico e isso auxilia o macanismo de busca. Em outras palavras é importante para ficar bem poscionado em em buscas do Google.

Essa tag funiciona da seguinte forma:

Comando: 


        <form method="get" action="cadastro.php">
            <label for="lnome">Nome:</label>
            <input type="text" id="lnome">
            <input type="submit" value="Enviar">
        </form>

Note que a palavra "lnome" é o que faz a ligação entre os dois elementos então ela devem ser iguaias e devem ser usada somente uma para cada label.

Resultado:

Note que agora ao clicar em nome automaticamente o cursor vai para a caixa de texto correspondente.

Tag input

Atributo name

Esse atributo name , também é usado dentro da tag input. Esse atributo serve para identificar a tag input, porém esse name será o título da coluna do formulário que o usuário preencher. Então esse name pode ser exatamente igual ao identificador da caixa de texto. 


        <form method="get" action="cadastro.php">
            <label for="lnome">Nome:</label>
            <input type="text" name="Nome" id="lnome" >
            <input type="submit" value="Enviar">
        </form>

Informação EXTRA

Para desativar o aquela aba de autopreenchimento dos formulários basta adicionar o atributo autocomplete e botar off:

OBS.: o ON vem ativado por padrão.

O comando é simples: 


        <form method="get" action="cadastro.php" autocomplete="off" >
            <label for="lnome">Nome:</label>
            <input type="text" name="Nome" id="lnome"
            <input type="submit" value="Enviar">
        </form>

Fieldset / Legend

Fieldset define um grupo de campos na forma de um fieldset. Por padrão, é exibida uma linha em volta do controle (pode ser personalizada via CSS) e permite usar a tag para descrever qual será o texto que identifica o grupo.

Veja como fica:

Ficha de Cadastro

Veja o código: 


<fieldset>

    <legend>Ficha de Cadastro</legend>
        
<form method="get" action="cadastro.php">

    <div>
        <label for="lnome">Nome:</label>
        <input type="text" id="lnome" required>
        
    </div>

    <div>
        <label for="lidade">Idade:</label>
        <input type="number" max="100" min="18" size="3" required>
    </div>

    <div>
        <input type="submit" value="Enviar">
    </div>

</form>
</fieldset>

type="radio"

Esse atributo de input é uma espécie de multipla escolha que permite apenas uma marcação, mas para isso devemos nos atentar para sempre usar os atributos a seguir corretamente:

  • name="": esse atributo deve ser usado sempre igual em todos os input relacionado(lembre que esse é o título da coluna do de resposta do formulário). Caso fique diferente ele irá marcar mais de umo opção.
  • value="": esse atributo também não pode faltar pois ele será a resposta que chegará na tabela do formulário.

Veja como fica:

Sexo:

Veja o código: 

    
        <fieldset>
        <legend>Sexo:</legend>
<form action="cadastro.php" method="get">
        <p>
            <label for="lmasculino">Masculino:</label>
            <input type="radio" name="sexo" id="lmasculino" value="Masculino">
        </p>
        <p>
            <label for="lfeminino">Feminino:</label>
            <input type="radio" name="sexo" id="lfeminino" value="Feminino">
        </p>
        <input type="submit" value="Enviar">
</form>
        </fieldset>

type="checkbox"

  • name="": Cada opção dessa checkbox equivale a uma coluna na planilha do formulário e esse coluna precisa ter um tútulo

type="range"

Basicamente exibe uma pequena barra horizontal com um controle deslizante. Mas possui atributos que auxiliam nas respotas que recebemos.

Veja como fica:

Nível de satisfação

Veja o código: 


<fieldset>
<form action="cadastro.php" method="get"> 
    <legend>Nível de satisfação</legend>

    <div>
        <label for="lsat">Escolha seu nível de satisfação:</label>
        <abbr title="Valor de: 1 a 5">
        <input type="range" id="lsat" name="Satisfação" min="1" max="5" value="5">
        </abbr>
    </div>
    <input type="submit" value"Enviar">
</form>
</fieldset>
Resumo de atributo
Tag Atributo Funcionalidade
form action Esse atributo dirá para onde os dados serão enviados e será ativado quando o atributo submit for acionado
method="get" Aparece no URL, não pode conter dados sensíveis, máximo 3kb e só envia texto
method="post" Dados não aparecem na URL, Aceita envios acima de 3kb e envia diversos tipos de arquivos.
autocomplete="off" Desliga o texto que auxilia o usuário (por padrão ele fica sempre ativado)
input type="submit" Cria um botão para enviar o formulário
type="reset" Cria um botão para apagar tudo que foi escrito no formulário inteiro
Value="Enviar" Nome que aparecerá no botão. Mas esse input também pode ser usado em caixa de texto, nesse caso ele já aparecerá um texto digitado, é muito usado em data já deixando um data atual pro usuário não ter que correr o calendário inteiro.
type="number" Caixa de texto só aceita número. Note que cria um seta para cima e para baixo dentro da caixa de texto para rolar os números
min="0" O VALOR MÍNIMO para enviar o formulário.
max="10" O VALOR MÁXIMO para enviar o formulário.
step="0.5" Só rola os números de 0,5 em 0,5
type="month" Abre um calendário no navegador para o usuário escolher mês e ano.
type="date" Igual month mas nesse inclui os dias(data completa) É importante sempre usar esse type para validar Informção antes de enviar. Isso evita erros de digitação do usuário.
type="time" Aceita somente horas e minutos
type="password" aparece "*" no lugar do que foi digitado
type="email" Informa o navegador que ali deverá ter um e-mail mas a validação não funciona tão bem pois ele só exige um @.
type="tel" Informa o navegador que ali deverá ter um telefone mas ele só exige um 8 dígitos e bota um traço depois do quarto número.
type="radio" Abre uma multipla escolha que permite só uma marcação. *Possui atributo obrigatório!
type="checkbox" Cria uma multipla escolha que permite mais de uma marcação. *Possui atributo obrigatório!
type="color" Abre uma caixa para seleção de cores e envia a cor usando código hexadecimal
type="range" Exibe como uma pequena barra horizontal com um controle deslizante. Pode usar os mesmos atributos da type="number" para melhorar as respostas.
pattern="estudar" Estudar mais sobre pattern pois é uma forma de validação em HTML. veja o pdf nas páginas 8 e 9
name="Nome" Esse name será o título da coluna do formulário que o usuário preencher.
required Exige que o formulário esteja preenchido para ser enviado
minlength="5 Exige que o formulário esteja preenchido com no mínimo 5 caracteres para ser enviado
maxlength="15 Exige que o formulário esteja preenchido com no máximo 15 caracteres para ser enviado
size="10" Define o tamanho da caixa para caber até 10 caracteres
placeholder="Digite seu nome aqui" Mostra uma dica dentro da caixa de texto(ela some assim que o usuário começar a digitar na caixa)
autocomplete="username" Guia o navegador para auxiliar o usuário com a sugestão correta, nesse caso seria o nome de Login. Para ver todas as opções basta pressionar"ctrl+espaço" entre as aspas. (Para usar esse autocomplete a tag form deverá estar com o autocomplete="ON")
autocomplete="current-password" O navegador irá sugerir a senha já cadastrada a senha recorrente
autocomplete="new-password" O navegador irá sugerir nada, ou o gerenciador de senha poderá sugerir uma senha forte e salvar nova senha.

Estudar mais expressões regulares, pesquisar por RegEx.Expressões regulares é para melhorar a validação dos dados enviados pelo usuário.

Tag select

Essa tag funiciona mais ou menos parecido com a type="radio" a diferença é que nessa as opções ficam recolhidas, por isso só é interessante usar a type="radio" quando tem até duas opções.

Essa tag utiliza os elementos a seguir

  • option: são as opções que apareceram para o usuário, Essa opção entra junto com o atributo value que é o que será enviado para o formulário.
  • optgroup divide a lista em grupos para facilitar a busca para o usuário e para isso temos que usar obrigatoriamente os seguintes atributos:
    • Value="": É importante botar value="" vazio, caso não coloque o usuário poderá selecionar essa opção.
    • label="xxx": Nesse elemento a label é usada como atributo. E nesse caso o xxx será o nome que aparecerá para o usuário na lista

Veja seu funcionamento abaixo:

Cadastro

Veja o código: 


<fieldset>
    <legend>Cadastro</legend>
<form method="get" action= "cadastro.php" autocomplete="on" >
    
    <div>
        <label for="lestado">Selecione seu Estado:</label> <br>
        <select name="Estado" id="lestado" required>
            <option value="" selected>Selecione um Estado</option>
            <optgroup value="" label="---Região Sudeste---">
                <option value="RJ" >Rio de Janeiro</option>
                <option value="SP">São Paulo</option>
                <option value="MG">Minas Gerais</option>
            </optgroup>
            <optgroup value="" label="---Região Sul---">
                <option  value="PR">Paraná</option>
                <option value="SC">Santa Catarina</option>
                <option  value="RS">Rio Grande do Sul</option>
            </optgroup>
        </select>
    </div>
            
    <input type="submit" value="Enviar">
    
</form>
</fieldset>

input com atributo list

O input com esse atributo funciona de forma parecida com a tag select com uma pequena diferença, o input aceita que o usuário escreva uma opção que não esteja listada.

Para isso usaremos o seguintes atributo:

  • list="lstxxx": Esse atributo não só faz o input trabalhar com uma lista como também faz o link com a lista que será usada por ele OBS.: o nome não pode ser o mesmo da tag label pois ele não consegue fazer o link. Um boa prática é sempre iniciar com lst

Tag datalist

Nessa tag é onde fica armazenada todas as opções que serão exposta para o usuário. Para isso ele usa o seguinte atributo e elemento:

  • id="lstxxx" Esse atributo é o que faz o link com o list do input, por isso esse nomes devem ser exatamente iguais.
  • option: Esse elemento funciona identificamente ao da tag select a única diferença é que aqui o value aparece para o usuário. o que estraga a apresentação. Porém como ainda não sei como o cadastro chega no php vou continuar usando dessa forma.
  • optgroup: O optgroup deveria funcionar, segundo a w3c. No entanto não funciona, então não use.

Veja seu funcionamento abaixo:

Cadastro do aquário

Veja o código: 


    <fieldset>
        <legend>  Cadastro do aquário</legend>

    <form action="cadastro.php" method="get">
    
        <label for="lpeixe">Escolha um peixe:</label> <br>
        <input type="text" name="peixe" id="lpeixe" list="lstpeixe">
            <datalist id="lstpeixe">
                <option value="neon">Neon</option>
                <option value="betta">Betta</option>
                <option value="guppy">Lebsite</option>    
            </datalist>
    
        <input type="submit" value="Enviar">

    </form>
    </fieldset>

Tag textarea

Essa tag cria um campo grande de mensagem, a grande diferença para o input type="text" é que aqui não tem limite de tamanho da mensagem. É importante por um max lengh mesmo que com o valor alto para que algum engraçadinho não derrube o site por causa de um texto gigante O usuário ainda pode ajustar o tamanho da área que está digitando o texto.

Veja um exemplo abaixo:

Avaliação do Produto

Veja o código: 


        <fieldset>
            <legend>Avaliação do Produto</legend>
        <form action="avaliacoes.php" method="post">
            <div>
                <p>
                    <label for="lnota">Nota:</label>
                    <abbr title="Nota de 1 a 5"><input type="range" name="Nota" id="lnota" min="1" max="5" value="5" ></abbr>
                </p>
            </div>
            <div>
                <p><label for="ltitulo">Título:</label></p>
                <p><input type="text" id="ltitulo" name="titulo"></p>
            </div>
            <div>
                <p><label for="ldescricao">Descreva sua experiência:</label></p>
                <p><textarea name="descricao" id="ldescricao" cols="30" rows="10" maxlength="1000"></textarea></p>
            </div>
            <p><input type="submit" value="Enviar"></p>
    
        </form>
    
        </fieldset>

Output

Como já é de imaginar a tag output não recebe a Informação do usuáio, mas ela envia informação para tela do usuário. Para isso precisamos usar um pouco de JavaScript, pois sem isso não tem como enviar alguma informção útil.

Veja abaixo um exemplo bem básico de output usando JavaScript onde ele mostra a nota que está sendo gerada pelo input range.

Avaliação do Produto

5

O comando: 


<fieldset>
    <legend>Avaliação do Produto</legend>
<form action="avaliacoes.php" method="post" oninput="inota.value=Number(lnota.value)">
    
    <div>
        <p>
            <label for="lnota">Nota:</label>
            <abbr title="Nota de 1 a 5"><input type="range" name="Nota" id="lnota" min="1" max="5" value="5" ></abbr>
        
            <output name="nota" id="inota">5</output>
        </p>
        </div>

</form>
</fieldset>

Note a importâcia do identificador id, todo id deve ser semântico para evitar confusões.
