Media Queries

Para estudar esse cáptulo será necessário utilizar uma extensão do google chrome que simula a tela de um celular no navegador.

Media Query é a capacidade do site de alterna entre diferenetes folhas de estilo dependendo do tipo de dispositivo com as Media types que está visualizado, e de algumas outras caractiriscas que veremos no decorrer do capítulo. Pelo que entendi é como se fosse uma picture e source media das folhas de estilos.

Como já vimos anteriormente, as folhas de estilos são criadas a em arquivo CSS e são linkadas no head com o comando: 


<link rel="stylesheet" href="../../modulo-01/capitulo-12/style.css">

Porém quando usamos media query, essas folhas são usadas de acordo com a tela ou modo saída da página. Veja algumas configurações disponíveis abaixo:

Existem as seguintes configurações de folha de estilo:

  • media="print": Modo impressora. Desse modo podemos ocultar alguns menus e outros itens da página desnecessários para impressão. Veja o comando abaixo:
    • <link rel="stylesheet" href="estilos/impressora.css" media="print">
  • media="screen": Modo Tela. Esse modo posui mais funções, mas veremos isso mais a frente. Veja o código do screen abaixo:
    • <link rel="stylesheet" href="estilos/tela.css" media="screen">
  • media="all" Configuração geral que se aplica a todos os dispositivos. Hierarquicamente é como se o all fosse a folha de CSS e o screen fosse a style feita no head do html
  • media="handheld": Modo dispositivos de mão
Exemplo de media="print"

media="screen"

O media screen ainda tem mais 2 adicionais que são os formatos de tela se são modo paisagem (como um notebook) ou modo retrato, como um celular ou tablet (nesse caso as tela também deitam mas gralmente a pessoas consomem conteúdo com a tela em pé mesmo)

Para cada um desses dois formatos de tela nós iremos usar um arquivo de CSS. E para referenciarmos cada um informando qual usar de acordo com o foramato da tela do usuário usaremos os seguintes comandos:

  • Modo retrato: Veja o código abaixo
    • <link rel="stylesheet" href="estilos/retrato.css" media="screen and (orientaion: portrait)">
  • Modo paisagem: Veja o código abaixo
    • <link rel="stylesheet" href="estilos/paisagem.css" media="screen and (orientation: landscape)">

Modo tudo em uma folha de css

Além dessa forma de criar várias folhas de de css temos ums opção de fazer tudo em apenas uma folha ou até mesmo no head do html usando style.

Para isso basta criar a regra (@) em style escrevendo tudo de media para frente de declaração que fazemos para link as folhas separadas. WHAT'S? . Veja o exemplo abaixo: 


<link rel="stylesheet" href="estilos/paisagem.css" media="screen and (orientation: landscape)">

Agora basta por @, remover o sinal de igual e a aspas, dessa parte marcada e por em uma folha de estilo única ou em style no head do html.

Exemplo na style: 


    @media screen and (orientation: landscape)

Mas qual opção é melhor?

Tudo vai depender do tamanho do projeto. Basicamente quanto maior o projeto maior a folha então nessa caso é melhor deixar separado para não ficar muito bagunçado. Já projetos menores dá pra fazer até no próprio head.

Conceito Mobile First

Mobile First foi um Conceito criado visando priorizar a visualisação de sites em dispositivos móveis. Isso não é a mesma coisa que criar um site responsivo para celular, isso significa criar um site para celular e deixa-lo responsivo para notebook e outra telas. Por isso o nome Mobile First.

Google

O criador desse conceito foi o chefe de produtos Google,(Luke Wroblewski) em 2009. Apartir de 2018 a Google passou a priorizar sites que aplicam esse conceito Mobile First, pois a quantidade de acessos por smartphones é muito superior aos acessos por notebook e outros dispositivos.

Principais Vantagens do Mobile First

  • Maior divulgação do site: pois como vimos anteriormente, ferramentas de busca vão valorizá-lo.
  • Melhor experiência do usuário: que vai ter acesso ao conteúdo que foi pensado para que pudesse ser consumido de forma mais confortável e sem confusões.
  • Aumento na credibilidade: os visitantes vão ter a percepção (ainda que inconsciente) de que quem criou o site se preocupou com a sua experiência.
  • Otimização do carregamento: uma vez que sofremos com conexões lentas e dispositivos populares sem tanto poder de processamento.

Mudança na prática

As principais diferenças de trabalhar usando o conceito Mobile First são:

  • Teste em celular: Toda página que criamos no VS Code vamos escrevendo o Código em verificando como ficou no navegador, mas apartir de agora faremos isso usando a tela de um celular através de alguma extensão no navegador que faça essa simulação
  • Media Query: Até aqui usamos o media query para adaptar o conteúdo do site para o celular e o "ALL" era baseado na tela de um notebook. Apartir de agora o "ALL" será baseado no celular e outros media query para outras telas
