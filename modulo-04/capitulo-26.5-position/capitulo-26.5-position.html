Display inline-block

Quando aplicamos o display inline block em um elemento que é naturalamente block, como uma div, ela deixa de se comportar como um bloco e passa a se comportar como uma palavra(na verdade palavra são naturalmente inline-block mas para fins de entendimento isso basta).

Isso quer dizer que os comando aplicado a textos funcionarão em div que estiverem como display inline-block.

Exemplos text-aling, center;

As div não ficam uma sob a outra quando usa word-spacing com valores negativos o máximo que acontece é ficarem coladas

Position Static

Static é o valor padrão da propriedade position. O box de um elemento com position:static é colocado na sua posição natural. Isso é: se for um box em display block ele fiacará colado ao lado esquerdo da tela e um embaixo deo outro, e si for inline block ele ficará um ao lado do outro.

Não podemos alterar a posição dos boxes dos elementos HTML com position: static.

Position Relative

O box de um elemento com position: relative é colocado em uma posição relativa a sua posição natural(static). As propriedades left, top, right e bottom são utilizadas para determinar o deslocamento entre a posição desejada e a posição natural.

O espaço que essa box ocuparia, se ele estivesse em sua posição natural, não poderá ser ocupado por nenhum outro elemento. E a box será projatada para uma posição em primeiro plano acima e será exibida por cima de qualquer outro elemento

Exemplo:


.conteiner{
    height: 400px;
    width: 400px;
    border: 2px solid #0f0;
}

.int-red-ex2{
    position: relative;
    top:200px;
    left:75px;
}

.int-green-ex2{
    position: relative;
    top:50px;
    left:50px;
}
.int-blue-ex2{
    position: relative;
    
}
div.int-green-ex2:hover{
    background-color: #000;
}

Observações

  • Espaço "físico": O espaço "físico" permanece no seu lugar natural. A única coisa que mudou de lugar foi sua projeção, sua visão.
  • Interações: As interações como cliques, ou efeitos ao passar o mouse, permanecem na sua projeção, pois ela se projeta por inteiro, exceto o seu espaço "físico".
  • Hierarquia: As projeções se projetam uma por cima da outra, o que define qual aparecerá por cima é ordem que estão sendo mencionadas no conteiner. Sendo a última a que ficará em por cima de todas e a primeira a que ficará por baixo de todas

Problemas desse posicionamento

  • Espaço em branco: Como o espaço natural não é ocupado por outro elemento. Ficará um espaço vazio no bloco o que pode ficar estéticamente feio ou estranho.
  • Sensível a modificações: Como esse posicionamento é projeta sua imagem para outro plano ele poderá gerar efeitos estranho caso seja feita qualquer modificação no futuro. Exemplo se mudar o tamanho do conteiner o bloco pode ir para cima de outro bloco, sair da tela, fica mal posicionado e etc..

Position: absolute;

Existem dois casos quando usamos o position absolute em uma div interna a um conteiner:

1º caso o conteiner tem um position definida como static. Ou não declarou position pois static é o padrão.

2º caso o conteiner possui uma position diferente de static.

Conteiner sem position definida

O bloco com position: absolute terá seu posicionamento relativo à página. Quando digo página não é body, nem html, mas literalmente a toda a extenção da página.

Exemplo de style: 

html{
    padding:50px;
}

body{
    padding: 50px;
}
div.conteiner{
    position: static;
}
div.int{
    postion: absolute;
    left: 0px;
}

Coteiner com position diferente de static

Caso o bloco possua um conteiner qualquer que o englobe com position diferente de static. O box absolute terá suas medições relativas a esse conteiner onde ele se encontra.

Exemplo style:


    div.conteiner{
        position: relative;
    }
    div.int-green-ex4{
        position: absolute;
        left: 10px;
    }
    div.int-blue-ex4{
        position: relative;
        left: 0px;
    }

Vale ressaltar que se fosse o conteiner "avô" com position diferente de static e o "pai" com position static. O bloco teria sua referência no conteiner "avô."

Observações

  • Absolute vs Relative: O conteiner absolute fica em baixo do relative caso estejam aparecendo no mesmo local da tela.
  • Projeta por inteiro: Diferentemente do relativo o posicionamento absoluto porjeta seu espaço "físico" junto com sua projeção visual. Isso significa que não ficará aquele espaço vazio indesjado no meio do conteiner.
  • Referência no posicionamento: se desloca relativo a qualquer conteiner que estiver contido desde que o position seja deferente de static.
  • Hierarquia de conteiner: Caso esteja dentro de mais de um conteiner diferente de static ele irá se referenciar por aquele que estiver mais próximo. Exemplo entre o "pai" e o "avô" a referência seria o "pai"
  • Hierarquia entre blocos Assim como os blocos de posicionamento relativo que se projetam uma por cima do outro, o que define qual aparecerá por cima aqui também é ordem que estão sendo mencionados no conteiner. Sendo o último o que ficará em por cima de todas e o primeiro o que ficará por baixo de todos

Como funciona o float

A propriedade float faz literalmente o bloco flutuar sobre o conteiner a ponto do conteiner perder sua referência.

Veja um exemplo abaixo:







o comando:


.conteiner-ex8{
    border: 2px solid #0f0;
    
}
.int-red-ex8{
    float:right;

}
.int-green-ex8{
    float: left;

}

.int-blue-ex8{
    float: left;
                    }

Por causa dessa característica de de perder sua referência, todas as box de um conteiner deverão ser float também. Caso contrário as boxes float iram sobrepor as demais.

Exemplo:

O comando:


<head>
    <style>
        
        .int-green-ex9{
        float: left;
        margin-left: 10px;
        }

    </style>
</head>
<body>
    <div class="conteiner-ex8">
        <div class="int int-red int-red-ex9"></div>
        <div class="int int-green int-green-ex9"></div>
        <div class="int int-blue int-blue-ex9"></div>
    </div>
</body>

Note que o box verde é o segundo no conteiner porém ele está por cima do azul. Isso aconteceu porque o verde "flutuou" com isso o conteiner e o azul perderam sua referência, então o azul subiu e o conteiner readaptou sua altura para enquadrar apenas as box vermelha e a azul.

Gambiarra

Existe um bug que faz o conteiner considerar os espaços ocupados pelas box que estiverem como float. Para isso basta usar o overflow:auto; no conteiner.

Digo bug pois o overflow auto obviamente não foi criado para essa finalidade.

Exemplo:

o comando:

.conteiner-ex10{
overflow: auto;
border: 2px solid #0f0;
}

Clear

Essa propriedade foi desenvolvida para trabalhar junto com float, pois sua finalidade é justamente fazer com que as boxes float reconheçam as que não são float. Fazendo assim que a float não sobreponha a elas.

Essa porpriedade funciona da seguinte forma:

  • Clear: left; Faz com que todas as float left reconheçam essa box. Porém as float right não reconhecem.
  • Clear: right; Faz com que todas as float right reconheçam essa box. Porém as float left não reconhecem.
  • Clear: both; Faz com que todas as float left e float right reconheçam essa box.

Exemplo:

O comando:


.conteiner-ex11{
    overflow: auto;
    border: 2px solid #0f0;
    }

    .int-red-ex11{
    float: left;
}
.int-green-ex11{
    float: left;
}
.int-blue-ex11{
    clear:left;
    display: inline-block;
}

Vale ressaltar que uma box pode ser clear ou float, ela não funciona com as duas ao mesmo tempo pois a clear perde a funcionalidade.

Curiosidade

Como toda box se alinha naturalmente a esquerda e a clear left faz as box float left reconhcerem a box clear left. Se usarmos a propriedade display: inline-box junto com clear left, essa box em questão se comportará como um float left.

Transform

A propriedade tranform também é uma propriedade de posicionamento porém ela tira como base suas próprias medidas. Função muito útil para evitar gambiarras como margem negativa.

Existem 6 tipos de transform e cada uma modifica o bloco ou img de uma forma. Essas 6 são:

  • transform: translate
  • transform: scale
  • transform: rotate
  • transform: skew
  • transform: matrix
  • transform: scale(0.5) translate(-100%, -100%); É possível usar mais de um trasform na mesma prorpiedade.

transform translate

Essa função translate funciona de forma parecida com a position relative, pois essa também se projeta sem mover o espaço físico dela. Entrentanto o translate usa medidas relativas a si próprio tipo as margens só que sem aumentar o espaço físico, pois esse fica parado igual ao position relative.

Um uso muito comum é usar essa propriedade com porcentagem afim de alinhas elementos, pois nela podemos alterar seus limites forçando a box a se guiar pelo centro invés das extremidades dela usando transform: translate(-50%, -50%); e com margin 50% é possível centralizar qualquer box.

A função translate funciona com eixo x e y onde entre () fica primeiro o eixo x e depois o y separados por vírgula.

Exemplo:

O Comando:


    .div-img-translate{
        transform: translate(-50%, -50%);
        margin:50%;	
    }

Observação

Não pode haver espaços entre a função e os () pois o navegador não reconhece o comando!!

transform scale

Essa função mexe na escala do largura (width) e altura (height) da box podendo esticar ou reduzir ambas em proporções independentes.

Porém depois de criar a escala, quando for mexer só no width ou só no height o box irá aumentar seguindo a proporção criada na escala.

Exemplo:

O comando:


.div-img-scale{
    transform: scale(2,0.5);  
}
  • Note que ao reduzir pela metade sua altura ela reduiu em direção a centro da box e não foi para o centro superior esquerdo como seria se uma box tivesse essa proporções
  • Não é necessário usar sempre dois valores nessa propriedade(um para o eixo X e outro para o Y). É possível também pôr apenas um valor e o progrma aplicará esse valor para os dois eixos.

transform: rotate

Como já é de se imagina essa função rotate rotaciona o elemanto ou a box.

Exemplo:

O comando:


.div-img-rotate{
    transform: rotate(180deg)
}

transform: origin

Essa propriedade modifica a origem da div. Origem seria o local da box que menos se movimenta ou que fica parada durante as trasformações da box.

Exemplo:

O comando:


.div-img-origin{
    transform-origin: right top;
    transform: rotate(270deg);
    
}

Observação:

  • Note que é possível relocar a origem da transformação tanto para o centro das laterais, quanto para as diagonais, para isso basta colocar o nome das laterais(left, right, top, bottom) que são unidas pela diagonal em questão.
  • Perceba também que mesmo a figura sendo circular, o navegador reconhece ela como quadrada.

Usando mais de uma transform

Para usar mais de uma propriredade transform, precisamos adicionar toda as transformações desejadas dentro da mesma propriedade trasform. Não é igual uma shothand, pois a shothand é opcional, já se quiser fazer mais de um tranformação no box só poderá fazer dessa forma pois, caso contrário, o CSS só reconhecerá a primeira ou nenhuma.

Exemplo:

O comando:


.div-img-mais{
    transform-origin: right top;
    transform:scale(.5,.3)translate(50%, 50%)rotate(180deg) ;
    border: 1px solid ;
}

Observações

  • A ordem das transformações faz toda a diferença no efeito final. Se atente ao que efeito desejado. Para isso faça o código passo a passo, mesmo que mentalmente, para notar o que irá acontecer na próxima transformação.

skew

skewx ou y

Essa transform atua inclinando os eixos x e y um de acordo com o outro como base, se inclinar o eixo x no ângulo de 60 graus ele abrirá 60º no verticeY(neg) com X(pos)

Exemplo:

O comando:


.div-img-skew{
    border: 1px solid ;
    width: 200px;
    background-color: #0f0;
    transform: skewX(60deg);
}

Observação

  • Se o ângulo se abre no verticeY(neg) com X(pos). Logo, se colocar 90º a box deverá sumir pois os eixos ficarão paralelos.
  • Se colocar o ângulo de 180º ele não espelha a imagem. Não entendi o porque. Deveria aparecer invertida
