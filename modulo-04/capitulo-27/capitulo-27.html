Flex Boxes

Flex box são caixas flexíveis. Valeu até a próxima!

A ideia inicial do uso das flex box é ajustar o conteúdo do site de acordo com cada formato e tamanho de tela disponível no mercado. Elas funcionam de seguinte forma:

A caixa externa dita as regras de tamanho disponível para exibição. Já as internas trabalham se moldando ao tamanho da externa de acordo com a regras que impulsermos a elas. O objetivo é que elas sempre trabalhem de forma fluida.

Para isso deixar as boxes com essa caracteríscas precisamos usar sempre 2 comandos:

  • display: flex; Display flex usamos na box externa. Mais comnhecida com conteiner
  • flex: auto; A flex auto usamos nas box internas. Caixas filhas.

Flex-direction

Flex-direction é um estilo usado na box conteiner. Como o nome já diz, ela serve para direcionar de onde para onde as box internas deverão se organizar.

Existem 04 direcionamentos possíveis:

  • flex-direction: row; Nessa forma as box internas iram se alinhar na horizontal da esquerda para a direita.
  • flex-direction: row-reverse; Nessa forma as box internas iram se alinhar na horizontal da direita para esquerda.
  • flex-direction: column; Nessa forma as box internas iram se alinhar na vertical de de cima para abaxo.
  • flex-direction: column-reverse; E nessa forma as box internas iram se alinhar na vertical de de baixo para acima.

Eixos (axis)

Além das direção da box conteiner, existe ainda mais dois eixos dentro dela:

  • main-axis Como o nome já diz esse é o eixo principal ele o eixo mais longo do retângulo. É eixo que atravessa o conteiner na direção da flex-direction.
  • cross-axis Também como o próprio nome já diz, essa cruza o perpendicularmente o eixo principal(main-axis).

Direcionamentos de cross-axis

Direcionamentos cross-axis são simples pois eles só quando a flex-direction está na horizontal(row) ou na vertical(column), veja:

  • O cross-axis, em casos de flex-directions horizontais(rows), Sempre serão de cima para baixo. Como o padrão column.
  • E quando a flex-direction está na vertical ele será sempre da esquerda para dereita. Como a row padrão.

Faça testes no head dessa página usando o primeiro exemplo.

flex-wrap: wrap;

Flex wrap significa encapsulamento flexível. Esse estilo é aplicado na box conteiner(externa). Isso significa que ela vai reorganizar as box internas para que elas caibam no formato da conteiner.

As box internas por sua vez vão se mudando de posição no sentido do cross-axis. Isso quer dizer que conforme for se estreitando a ultima no sentido da flex-direction irá mudar de linha ou coluna no sentido do eixo axis

Wrap, Nowrap e Wrap-Reverse

  • NoWrap: Por padrão todo conteiner é nowrap, isso significa que as box internas vão se expremer até seus conteúdo permitirem, qd não for mais possível ajustar seus conteúdos ela não irá se ajustar mais e começará a gerar overflow

    • Comando:

                flex-wrap: nowrap;
  • Wrap: Esse comando divide o conteiner criando linhas paralelas a main-axis no sentido de cross-axis. Na prática, esse comando faz com que a última box interna vá para linha ou coluna a direção da axis

    • Comando:

                flex-wrap: wrap
            ;
  • Wrap-Reverse: Esse comando divide o conteiner criando linhas paralelas a main-axis no sentido contrário da cross-axis. Na prática, esse comando faz com que a última box interna vá para linha ou coluna oposta a direção da axis

    • Comando:

                flex-wrap: wrap-reverse;
  • Flex-flow: Flex-flow é uma espécie de shothand da flexbox, onde primeiro definimos a flex directions e depois a flex-wrap. Inclusive a W3C recomenda que sempre use flex-flow.

    • Exemplo de comando no head:

                flex-flow: column wrap

Anatomia dos itens flex-box

Por padrão os itens do conteiner não são flexíveis, isso quer dizer que eles apenas obedecem a organição do conteiner. Entretanto, tem como mudar o comportamento individual de cada item para que cada um se organize do seu jeito dependendo do formato do conteiner.

order

Order é uma forma de organizar os itens dentro de um conteiner. Por padrão todo item do conteiner recebe order=0, sendo assim a ordem que escrevemos a div dentro do conteiner é a ordem que os itens ficarão organizados. Porém nem sempre isso é viável dependendo do tamanho do código, por isso exite o order

Exemplo

1
2
3
4
5

Comando: 


<div class="contex19">
    <div class="intex" style="order: 5">1</div>
    <div class="intex" style="order: 4">2</div>
    <div class="intex" style="order: 3">3</div>
    <div class="intex" style="order: 2">4</div>
    <div class="intex" style="order: 1">5</div>
</div>

align-self

Essa propriedade de estilo tem como principal finalidade dar autonomia as box internas de uma conteiner. Até aqui vimos todas as boxes trabalhando iguais, apenas seguindo as regras do conteiner. Mas agora com essa propriedade poderemos dar estilos independentes para as box internas.

Para isso usaremos 5 estilos:

  • auto ou inehit: Esse estilo é o padrão, portanto ele seguirá as regras do conteiner
  • flex-start: Esse alinhamento faz com que a box vá para perto do início da cross-axis
  • flex-end: Esse, naturalmente, faz a box ir para o final da cross-axis
  • center: Centraliza a box na metade da cross-axis. em outras palavras põe a box em cima do eixo main-axis.
  • stretch: estica a box de uma ponta a outra da cross-axis

flex-basis

Essa propriedade gera larguras flexíveis e ela trabalha de duas formas diferente dependendo das regras do conteiner.

  • Conteiner com flex-wrap: wrap; Nesse caso ela irá manter a largura definida pela flex-basis mesmo que o conteiner fique menor, pois ela irá quebrar as linhas até que todas as box estejam em coluna e só então, quando começar a estreitar abaixo da largura definida pela flex-basis, ela irá reduzir seu tamanho:
  • Conteiner com flex-wrap: nowrap; Nesse caso box interna irá estreitar assim que o conteiner reduzir seu tamanho abaixo da largura definida pela flex-basis.

flex-grow & flex-shrink

Flex-grow e Flex-shrink são propriedades que trabalham juntas com a flex-basis, pois elas habilitam ou desabilitam a flexbilidade da largura definida pela flex-basis.

  • flex-grow: Por padrão ela fica desabilitada. Quando essa propriedade está habilitada ela permite que a box interna aumente, a largura definida pela flex-basis,para cobrir todo o excesso da conteiner.

    • Comando: 

        
flex-grow: 1;
flex-grow: 0;
  • flex-shrink: Essa propriedade vem habilitada por padrão. Quando ela está habilitada ela permite que a box interna reduza, a largura definida pela flex-basis, para caber no conteiner.

    • Comando: 

    flex-shrink: 1;
flex-shrink: 0;

Outro ponto importante dessas duas propriedades é que, elas trabalham com valores proporcionais dentro do mesmo conteiner. Isso quer dizer que, se o valor de uma box é 1 e outra é 2, a box de valor dois aumentará ou reduzirá o dobro da box com valor 1.

O cálculo

Em caso que contenham varias box com valores diferentes dentro do mesmo conteiner o cálculo será feito da seguinte forma:

A box com o menor valor acima de 0 servirá como base para calcular a proporção de crescimento das demais. Exemplo:

Se uma box tem valor 1 outra tem valor 2, a 2 crescerá duas vezes mais. Até aqui ok.

Porém se duas box têm o valor 2, as duas crecerão iguais, pois a proporção aqui é de 1 para 1

Agora se uma tem valor 2 outra, 3 e outra 4?

  • A 2 crescerá normal
  • A 3 crescerá uma vez e 1/3 a mais que a box de valor 2
  • E a 4 crescerá o dobro da box de valor 2, pois a 2 é o valor base por ser o menor.

Veja esse exemplo abaixo:

flex-gorw:2
flex-gorw:3
flex-gorw:4

O mesmo também vale para o flex-shrink. Veja:

flex-shrink:2
flex-shrink:3
flex-shrink:4

Propriedade flex

A propriedade flex funciona como uma shothand. Assim como a flex-flow é usada no conteirner, a flex é usada nas box internas.

Inclusive a W3C recomenda que usemos sempre a propriedade flex para encurtar os comandos assim com ela também recomenda o uso da flex-flow.

Assim como toda "shothand", essa propriedade tem uma ordem de estilos. Veja a abaixo:

  1. flex-grow
  2. flex-shrink
  3. flex-basis

O comando: 

         flex: 2 1 100px

Veja agora as aplicações mais comuns desse comando flex.

Estilos Funcionamento e aplicações
flex:0 1; São dois comando que Resetam a box para os padrões de fábrica digamos assim. isso é usado para evitar que algum estilo geraizado interfira no funcionamento esperado pela box interna..
flex: initial
flex: 0 0 auto; Esses comando removem a flexibilidade da box e deixa ela apenas no tamanho do conteúdo delas.
flex: none;
flex: 1 1 auto; Esses comandos fazem o inverso do acima. Esse aqui deixa a box 100% flexível e ajusta seu conteúdo interno também.
flex: auto;
flex: 1; Quando tem apenas um número significa que só foi definido o flex grow. Logo o flex shrink e o flex-basis são o valor padrão. Seria o mesmo que escrever flex: 2 1 auto;
