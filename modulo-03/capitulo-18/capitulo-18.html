Git

GitHub é um dos Sistema de Controle de Versões (do inglês Version Control System - VCS), mais famosos que existe foi feito por Linus Torvalds, o mesmo criador do núcleo Linux.

O Git é totalmente focado no nosso computador. Ele cria uma pasta local que fica oculta na pasta do projeto que desejamos "salvar as versões". Nessa pasta ele salva todas a modificações que fazemos nos projetos.

A grande vantagem de usar um sistema de controle de versões como o Git é poder voltar a qualquer momento para qualquer versão anterior do projeto de forma imediata, tudo 100% transparente para o programador. Dessa forma caso cometa alguma "cagada" no projeto, você pode simplesmente voltar para versão anterior a "cagada". É como fazer o mega "CTRL+Z"

GitHub

GitHub nada mais é que o Git on line, porém não tão simple. O GitHub trabalha junto com o Git pegando os versionamentos locais(que você criou com o Git) e jogando para a nuvem em sua conta. Entretanto ele faz mais que isso pois além dele garantir que você não perca seu projeto caso aconteça algo com sua máquina, ele também permite que vocé trabalhe com mais de uma pessoa no projeto cada um em sua máquina.

Existe ainda um recurso chamado GitHub Pages, que vai permitir a hospedagem gratuita de sites simples, que usem HTML + CSS + JS e disponibilize o acesso através de uma URL.

Criando novas versões

Apartir de agora( Depois de instalados, criado conta e adicionado os arquivos ao repositório) Sempre que fizer uma modificação no VSCode, ele irá mostrar um M ao lado do arquivo. Isso é i Git avisando que foi feita uma modificação naquele arquivo e que precisa ser salva uma nova versão.

Para isso basta ir no app GitHub → abrir o arquivo modificado → ir em commit to master → Digitar um título e uma breve descrição do que foi feito→ e clicar em Commit to master.

Pronto agora você acabou de criar um versão atualizada da página no seu repositório LOCAL. Agora precisa atualizar o seu reposiório REMOTO.

Para isso basta ir em GitHub e → Push Origin. isso salvará sua nova versão no GitHub(na nuvem)

Modificar de privado para público e vice versa

Ao clicar em Push Origin a primeira vez ele perguntará se o arquivo será público ou privado caso posteriormente queira modificar essa opcão basta abrir o site GitHub na sua conta → repositories→selecionar o repositório que deseja modificar→ Settings → desce a página até Danger Zone → Change repository visibility → clicar em Change visibility.

Como apagar repositório local

Para apagar o repositório local basta:
Abrir o GitHub→ selecionar o repositório que deseja apagar em "Current repository"→Clicar na aba "Repository"→ e Remove.

Dessa forma você apagará o repositório LOCAL.

Como apagar repositório remoto

Para apagar o repositório Remoto basta:
Abrir o site GitHub na sua conta → repositories→selecionar o repositório que deseja apagar→ Settings → desce a página até Danger Zone → Clicar em "Delete this repository"

GitHub Pages

O recurso do GitHub Pages não está ativo por padrão em nossos repositórios remotos, mas é muito fácil de habilitar. Isso só pode ser feito diretamente no site do GitHub, individualmente para cada repositório que queremos usar para hospedar projetos HTML.

Abra seu perfil no site do GitHub → abra o repositório que quer habilitar a hospedagem → clique sobre a aba Settings e role a tela até a área GitHub Pages.

Na área de opções, existem duas maneiras de habilitar o GitHub Pages:

Hospedagem gratuita

Na primeira, clique no botão None e escolha a opção master branch. Com essa maneira, você vai precisar ter um arquivo index.html com o código da página principal que será exibida ao acessar o site. Essa técnica é mais usada quando queremos hospedar um site único e receber uma URL para apresentar o projeto a outra pessoa.

Hospedar vários projetos

A segunda maneira é clicando sobre o botão Choose a theme, que vai permitir escolher um tema para exibir o conteúdo do arquivo README.md como uma página. Essa segunda técnica é mais usada quando vamos hospedar vários projetos em um mesmo repositório ou quando vamos guardar informações sobre um projeto que não é um site, como download de instaladores e coisas do tipo.
