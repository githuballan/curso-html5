Git

Git é um dos sistemas de controle de versões (Version Control System - VCS) mais famosos, criado por Linus Torvalds, o mesmo criador do núcleo Linux.

O Git é totalmente focado no nosso computador. Ele cria uma pasta local oculta na pasta do projeto, onde salva todas as modificações feitas nos arquivos.

A grande vantagem de usar um sistema de controle de versões como o Git é poder voltar a qualquer momento para qualquer versão anterior do projeto de forma imediata, tudo 100% transparente para o programador. Dessa forma, caso cometa algum erro no projeto, você pode simplesmente voltar para a versão anterior. É como fazer um mega "CTRL+Z".

Assista aos primeiros passos com GitHub nos quatro primeiros vídeos deste módulo.

GitHub

GitHub é uma plataforma online que trabalha junto com o Git, permitindo que você armazene seus versionamentos locais na nuvem. Além de garantir que você não perca seu projeto caso algo aconteça com sua máquina, ele também permite que você trabalhe com outras pessoas no mesmo projeto, cada uma em sua máquina.

Existe ainda um recurso chamado GitHub Pages, que permite a hospedagem gratuita de sites simples, que usem HTML, CSS e JavaScript, disponibilizando o acesso através de uma URL.

Criando novas versões

A partir de agora, sempre que fizer uma modificação no VSCode, ele mostrará um "M" ao lado do arquivo. Isso é o Git avisando que foi feita uma modificação naquele arquivo e que precisa ser salva uma nova versão.

Para isso, basta ir no app GitHub, abrir o arquivo modificado, ir em "Commit to master", digitar um título e uma breve descrição do que foi feito, e clicar em "Commit to master".

Pronto, agora você acabou de criar uma versão atualizada da página no seu repositório local. Agora precisa atualizar o seu repositório remoto.

Para isso, basta ir em GitHub e clicar em "Push Origin". Isso salvará sua nova versão no GitHub (na nuvem).

Modificar de privado para público e vice-versa

Ao clicar em "Push Origin" pela primeira vez, ele perguntará se o arquivo será público ou privado. Caso posteriormente queira modificar essa opção, basta abrir o site do GitHub na sua conta, ir em "Repositories", selecionar o repositório que deseja modificar, ir em "Settings", descer a página até "Danger Zone" e clicar em "Change repository visibility".

Como apagar repositório local

Para apagar o repositório local, basta abrir o GitHub, selecionar o repositório que deseja apagar em "Current repository", clicar na aba "Repository" e selecionar "Remove".

Dessa forma, você apagará o repositório local.

Como apagar repositório remoto

Para apagar o repositório remoto, basta abrir o site do GitHub na sua conta, ir em "Repositories", selecionar o repositório que deseja apagar, ir em "Settings", descer a página até "Danger Zone" e clicar em "Delete this repository".

GitHub Pages

O recurso do GitHub Pages não está ativo por padrão em nossos repositórios remotos, mas é muito fácil de habilitar. Isso só pode ser feito diretamente no site do GitHub, individualmente para cada repositório que queremos usar para hospedar projetos HTML.

Abra seu perfil no site do GitHub, abra o repositório que quer habilitar a hospedagem, clique sobre a aba "Settings" e role a tela até a área "GitHub Pages".

Na área de opções, existem duas maneiras de habilitar o GitHub Pages:

Hospedagem gratuita

Na primeira, clique no botão "None" e escolha a opção "master branch". Com essa maneira, você vai precisar ter um arquivo index.html com o código da página principal que será exibida ao acessar o site. Essa técnica é mais usada quando queremos hospedar um site único e receber uma URL para apresentar o projeto a outra pessoa.

Hospedar vários projetos

A segunda maneira é clicando sobre o botão "Choose a theme", que vai permitir escolher um tema para exibir o conteúdo do arquivo README.md como uma página. Essa segunda técnica é mais usada quando vamos hospedar vários projetos em um mesmo repositório ou quando vamos guardar informações sobre um projeto que não é um site, como download de instaladores e coisas do tipo.
