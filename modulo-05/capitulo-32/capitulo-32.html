Funções

Funções são blocos de comando com um nome pra vc chamar quando precisar executar aquela série de comandos sem precisar ficar toda hora redigitando tudo. Tipo a formula soma do excel, você pode ficar digitando + um monte de vezes ou simplesmente usar a fórmula soma.

Nomeclaturas que precisamos conhecer:

  • Parâmetros: são variáveis equivalente as declaradas com "var". Porém declara-las dessa forma fica muito mais fácil.

    • Observação

    Esses parâmetros podem ter um valor padrão inicial ou não e, assim como as variáveis, é aconselhável sempre ter um valor padrão em todos eles, afim de evitar bugs. Para declara-los com valor padrão basta fazer igual fazemos quando declara variável, usando o sinal de "=" para atribuir o valor padrão.

  • Argumentos: são os valores atribuidos aos parâmetros da função quando ela é chamada.

Funções Declaradas

Uma função declarada é definida usando a palavra-chave "function", seguida pelo nome da função, uma lista de parâmetros entre parênteses e o corpo da função entre chaves.

Você pode chamar uma função pelo seu nome, seguido por parênteses contendo quaisquer argumentos necessários.

function soma(n1=0,n2=0,n3=0,n4=0)
    {
        return n1+n2+n3+n4;
    };

console.log(soma(10,15,5,7));//37

Informação relevantes

  • Se faltasse o valor de alguma variável exemplo:"soma(10,15,5)" nesse caso resultaria 20. Mas apenas porque o valor da variável padrão é zero, caso não houvesse declarado um valor padrão isso resultaria um erro.
  • Uma função declarada pode ser chamada em qualquer momento mesmo antes de declara-la. Pois elas são tipo (hoisted), basisicamente significa que elas são carregadas primeiro pelo JS.
  • Mesmo que a função não possua argumentos ou parametros ela sempre precisará te os () mesmo que vazio, tanto na hora que criar a função quanto na hora que for chamar a função para exibir seu resultado. Caso crie a função sem os parenteses o JS acasará um ERRO e caso chame a função sem os () o JS exibirá todo código da função escrito.

Função Expressa ou Função anônima

Uma função expressa é uma função que pode ser armazenada em uma variável. Ao contrário das funções declaradas, as funções expressas não são elevadas (hoisted), ou seja, você não pode chamá-las antes de serem definidas.

const soma1 =function (n1=0,n2=0,n3=0,n4=0){
    return n1+n2+n3+n4;
};

console.log(soma1(5,5,15,8));//33

Repare que o nome da função so tornou o nome da variável e a função se tornou o valor padrão da variável

Note também que a variável usada foi "const", pois não podemos alterar o valor dela (nesse caso a função) posteriormente.

Funções de Flecha (Arrow Functions)

Introduzidas no ES6 (versão do JavaScript lançada em 2015), as funções de flecha têm uma sintaxe mais curta e não têm seu próprio "this"(a palavra-chave function)

É mais parecida com a função expressa, pois também é criada dentro de uma variável.

const soma2=(a1=0,a2=0,a3=0,a4=0)=>a1+a2+a3+a4;

console.log(soma2(5,5,18,8));//36

Parâmetro REST em funções

O parâmetro rest (ou rest parameter) é uma funcionalidade do JavaScript que permite representar um número indefinido de argumentos como um array. Ele é útil quando você precisa criar funções onde o número de argumentos possam variar.

O parâmetro rest é indicado por três pontos (...) antes do nome do parâmetro na definição de uma função. Quando você usa o parâmetro rest, todos os argumentos passados para a função, após os 3 pontos, serão colocados em um array.

Exemplo de aplicação do parametro REST

function soma(...valores){
    let tam=valores.length
    let res=0
    for(let i=0;i<tam;i++){
        res+=valores[i]
    }
    return res
}
console.log(soma(10,3,2,5)) // 20
