Estruturas Condicionais

if e else

If e else funciona como if else da linguagem C, C++, é tudo igual, só muda a forma de escrever.

O comando if permite que valores booleanos sejam testados, funciona parecida com o ternário que vimos anteriormente. Se o valor passado como parâmetro para o comando if for true, o bloco do if é executado. Caso seja false ele executará o else.

Vale ressaltar que o bloco de comandos if ou else, só será lido caso a condição seja satisfeita caso contrário o programa irá pular o bloco inteiro.

Exemplo:

if(a==b) {c=1}
else{c=2}

Nesse caso o resultado será 2.

  • Não é necessário abrir o bloco de comando(as chaves) no exemplo acima, pois só tem um resultado simples, o bloco de comando só é necessário quando o programa tem que executar mais de uma ação.
  • No exemplo acima poderia ter sido usado o operador ternário para simplificar o código.

É possivel impor mais de uma condição no mesmo if usando o else if.

Exemplo:

if(1+a==3) {c="certo"} 
else if(1+a==4) {c="quase"}
else {c="errado"}

Switch

Switch significa escolha em inglês e é exatamente o que ele faz. Seu funcionamento é parecido com o if, onde ele testa um determinado resultado. Porém, ele faz vários testes internos até que um caso funcione e ele execute o que for estabelecido.

Entretanto, ele não executará apenas a ação do resultado que retornou verdadeiro; ele executará todas as ações posteriores àquele resultado que retornou verdadeiro, até o comando break. Esse comando é o que interrompe o switch.

Outra ressalva importante é que o switch não faz testes comparativos como ">", "<", "==", "!=", etc., igual ao if; ele apenas verifica se o resultado está igual ao que for estabelecido no case.

Exemplo:

switch(a){
    case 2:
        console.log("case 2")
        break

    case 3:
        console.log("case 3")
        break

    case 4:
        console.log("case 4")
        break

    default:
        console.log("default")

}

Note que "a" é uma variável e ela terá seus valores testados em cada case.

Caso a fosse igual a 7, por exemplo, o resultado seria default pois não existe case on 7 retorne verdadeiro.

Se não tivesse usado break e "a" fosse igual a 4, a função switch retornaria 4 e default. Por isso é importante usar o break, para evitar resultados inesperados e consequentemente bugs.

Estrutura de loop/repetição

Funções de loop são utilizadas quando precisamos que algo se repita até que seja satisfeita alguma condições. Como acertar o login e a senha para entrar no site, ou fazer um cálculo de potênciação ou raiz onde precisamos multiplicar ou dividir várias vezes o mesmo número.

Para isso exsitem duas funções muito usadas em JS que são:

  • While
  • For

É importante saber que na função loop é muito fácil criar um bug que travará todo o site talves até a máquina do usuário, são os chamados loop infinitos onde você impõe uma resultado inalcansável e a máquina ficará repetindo o loop até esgotar toda a memória RAM ou superaquecer o processador. Por isso muito cuidado e sempre teste seus loop antes de enviar para o site.

Para evitar isso, certifique-se de que a condição do loop eventualmente se torne falsa. Para isso podemos por o comando break dentro de uma função no loop de forma que ele possa seja acionado caso o loop atinja certo número de repetição evitando problemas, pois assim como em switch o break interromperá o loop.

While

While significa enquanto, isso quer dizer que ela rodará o loop enquanto determinada condição não for satisfeita e assim que for satisfeita ela sairá do loop e seguirá para o resto do programa.

A estrutura básica dessa função é apenas uma condição seguida do seu bloco de comandos. semelhante ao if.

Por só existir essas duas condições é muito importante que primeiro crie o bloco de comando já com a função que satisfaça a condição e depois a condição que deverá ser satisfeita para não travar a máquina no looping infinito ao testar.

ATENÇÃO

É importante desliga o live server enquanto criar o while, pois no porcesso a página irá carregar o programa inacabado e travará em algum momento por causa do looping infinito.

Outro ponto importante, é sempre deixar um break quando for testar o programa, para não precisar ficar travando a página toda vez que der um erro.


var backup =1;
var w=1;
while(w<11){
    console.log(w+"ª vez");
    w++;

    backup++;
    if(backup>20){console.log("break"); break}
}

Se w fosse declarado igual a 12 o programa iria pular o while.

A forma correta de ler a função while é: executar o bloco de comando enquanto w for menor que 11.

do... while

O while verifica a condição antes de executar o bloco de código, enquanto o do...while executa o bloco de código pelo menos uma vez antes de verificar a condição.

Exemplo:

var d=11;
backup =1
do{

console.log(d+"ª vez");
d++;


backup++
if(backup>200){console.log("break"); break}
}while(d<10);

No exemplo acima a ação foi executada uma vez e saiu do loop, pois a condição já iniciou satisfeita.

For

O comando for em JavaScript é utilizado para executar um bloco de código repetidamente, geralmente com base em uma condição que é verificada antes de cada iteração.

Esse comando possui uma estrutura básica composta por 3 partes:

  1. Inicialização: Executada uma vez antes do início do loop. Geralmente é usada para definir e inicializar a variável de controle do loop.
  2. Condição: Avaliada antes de cada iteração do loop. Se a condição for verdadeira, o bloco de código dentro do loop é executado. Se for falsa, o loop termina.
  3. Incremento: Executado após cada iteração do loop. Normalmente, é usado para atualizar a variável de controle do loop.

Veja um exemplo:

for(var i=1; i<6; i++){
    console.log(i+"ª vez impresso")
}

Omissão de Componentes: É possível omitir um ou mais componentes do loop, embora isso possa tornar o código menos claro esujeito a loop infinito. Por exemplo é possível não decalarar a condição ou o incremento deixando o espaço em branco, entretanto é necessário por os ";".

Uma forma de evitar loop infinito é criando uma sistema de segurança como backup na hora de testar o código para isso basta decelarar uma variável antes e fora do loop e dentro do loop incrementar essa variável logo após o incremento criar i um if var > que 100 break. Dessa forma evitará erros no teste.

Exemplo:

var backup =1
for(var i=1; i<31;i++){
    document.write(i+"ª vez impresso<br>")
    backup++   
    if(backup>400){document.write("break"); break;}
}

Estrutura Math.random

Essa função não está relacionada a controle de fluxo.

A função Math.random gera números decimais aleatórios cada vez que ela é executada.

Veja um exemplo:

var m=Math.random()*100;

Note que o valor está sendo multiplicado por 100. Isso ocorre porque os números gerados são sempre decimais.

Importante ressaltar que o "M" de Math.random tem que ser maiúsculo.
